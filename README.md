# HealthVault

> **Персональная база знаний о здоровье, питании, спорте и добавках**

---

## 📋 Основные файлы

- **`HEALTH.md`** - Профиль здоровья, спорт, питание, добавки
- **`KNOWLEDGE_BASE.md`** - База знаний по медицинским анализам
- **`ARCHITECTURE.md`** - Архитектура проекта (база знаний vs контекст)
- **`ONBOARDING.md`** - Руководство для разработчиков (настройка, ключи, запуск)
- **`QUICK_START.md`** - Быстрый старт для нового разработчика
- **`RESTORE_BACKUP.md`** - Инструкция по восстановлению бэкапа
- **`data/nutrition/nutrition_log.json`** - Дневник питания и добавок
- **`data/workouts_database.json`** - База тренировок

---

## 🚀 Ежедневное использование

Основной интерфейс - **Telegram-бот** (`telegram-bot/`). Через него можно:
- Добавлять питание и добавки
- Просматривать данные
- Анализировать метрики
- Обновлять данные из различных источников

---

## 🔄 Обновление данных

Данные обновляются через Telegram-бота или по запросу. Скрипты обработки:
- `scripts/garmin/download_garmin_data.py` - загрузка данных Garmin
- `scripts/apple-health/parse_health_export.py` - парсинг Apple Health
- `scripts/sleepcycle/parse_sleepcycle_export.py` - обработка SleepCycle (автоматически находит файл в Downloads)

## 🛠️ Разработка и Тестирование

В проекте настроена автоматическая система контроля качества ("Защита от поломок").

### Автоматический запуск
При запуске бота через команду `make run` автоматически выполняются проверки:
1. **Тесты (`pytest`)**: Проверяют кейсы с витаминами, весом порций (пицца) и дубликатами.
2. **Типы (`mypy`)**: Проверяет код на ошибки типов данных.

Если проверки не проходят, бот **не запустится**. Это защищает от случайных поломок.

### Ручные команды
- `make test` — Запустить только тесты (без старта бота).
- `make check-types` — Запустить только проверку типов.
- `make install` — Установить зависимости (включая инструменты тестирования).

> **Для разработчиков (Cursor/Windsurf)**: Перед тем как считать задачу выполненной, обязательно запускайте `make test`, чтобы убедиться, что вы ничего не сломали.

## 🔒 Безопасность

- Все чувствительные данные (пароли, API ключи) хранятся в `.env` файле или отдельных файлах ключей
- Файлы с ключами исключены из Git (`.gitignore`):
  - `.env`
  - `.google_vision_api_key`
  - `.openai_api_key`
- Репозиторий приватный

**⚠️ Для нового разработчика:** См. `ONBOARDING.md` — там описано, где хранятся ключи и как их получить.

## 💾 Бэкапы проекта

Перед передачей проекта другой системе или перед крупными изменениями рекомендуется создать бэкап.

### Создание бэкапа:

```bash
cd HealthVault
./scripts/create_backup.sh
```

Бэкап сохраняется в `~/backups/HealthVault_backup_YYYY-MM-DD_HH-MM-SS.tar.gz`

**Что включается в бэкап:**
- ✅ Исходный код (telegram-bot/, scripts/)
- ✅ Базы знаний (knowledge_base.json, nutrition_log.json, workouts_database.json)
- ✅ Документация (все .md файлы)
- ✅ Конфигурация (.gitignore, requirements.txt)

**Что НЕ включается:**
- ❌ Большие файлы данных (PDF анализов, фото, данные Garmin)
- ❌ Временные файлы (venv/, __pycache__/, logs/)
- ❌ Секреты (.env, API ключи)

### Восстановление бэкапа:

См. подробную инструкцию в **`RESTORE_BACKUP.md`**

**Кратко:**
1. Найти нужный бэкап: `ls -lh ~/backups/HealthVault_backup_*.tar.gz`
2. Распаковать: `tar -xzf ~/backups/HealthVault_backup_YYYY-MM-DD_HH-MM-SS.tar.gz`
3. Восстановить файлы (см. `RESTORE_BACKUP.md` для деталей)
