# HealthVault

> **Персональная база знаний о здоровье, питании, спорте и добавках**

---

## 📋 Основные файлы

- **`HEALTH.md`** - Профиль здоровья, спорт, питание, добавки
- **`KNOWLEDGE_BASE.md`** - База знаний по медицинским анализам
- **`ARCHITECTURE.md`** - Архитектура проекта (база знаний vs контекст)
## 🚀 Использование (Только Сервер)

Проект работает **только** на выделенном сервере в Нидерландах (`146.103.111.109`).
Локальный запуск не поддерживается.

Основной интерфейс - **Telegram-бот** (`telegram-bot/`).

---

## 🚑 Управление процессами

**После каждого обновления кода** нужно задеплоить и перезапустить бота на сервере (иначе будет крутиться старая версия):

```bash
./deploy.sh
```

Скрипт: загружает код на сервер → пересобирает Docker-образ → перезапускает контейнеры. Подробнее: `docs/DEPLOYMENT.md`.

**Диагностика:**
```bash
./scripts/diagnose_server.sh
```

**Перезапуск бота:**
```bash
ssh root@146.103.111.109 "cd /root/healthvault && docker-compose down && docker-compose up -d --build"
```

## 📚 Справочник данных (Data Guide)

Где искать ответы на вопросы о здоровье, анализах и метриках.

### 1. Накапливаемые данные (Сервер/Бот)
Эти данные приходят через Telegram-бот и сохраняются в SQL базе данных (PostgreSQL).

| Тип данных | Таблица БД | Описание |
|------------|------------|----------|
| **Вес и состав тела** | `weights` | Данные с умных весов (скриншоты, Zepp Life) и Apple Health.<br>✅ Вес, % жира, **висцеральный жир**, мышцы.<br>❌ Apple Health часто не хранит висцеральный жир, он берется из OCR скриншотов. |
| **Питание** | `nutrition_log` | Ежедневные записи еды + фото блюд. Подсчет КБЖУ. |
| **Витамины** | `supplements_log` | Лог приема добавок (фактическое время приема). |
| **Активность** | `activity_log` | Шаги, калории, сон.<br>Синхронизируется с Garmin (через скрипт) и Apple Health. |

### 2. База знаний (Файлы / HealthVault)
Статичные данные, документы и экспорты. Лежат в папке `HealthVault/data/`.

| Папка / Файл | Содержимое |
|--------------|------------|
| `KNOWLEDGE_BASE.md` | **ГЛАВНЫЙ ИНДЕКС**. Сводная таблица всех анализов с 2009 года. Группа крови, динамика гормонов, витаминов. |
| `HEALTH.md` | **Профиль здоровья**. Текущие цели, привычки, диагнозы, "ground truth" (текущий вес, возраст). |
| `data/blood-tests/` | PDF файлы анализов крови (исходники). |
| `data/medical-records/` | Заключения врачей, ЭКГ, результаты обследований. |
| `data/apple-health/` | Экспорты XML из Apple Health (исторические данные). |
| `data/garmin/` | Выгрузки из Garmin Connect (пульс, тренировки, сон). |
| `data/workouts_database.json` | История тренировок (Кроссфит, Бег). |

### 3. Где искать ответ на вопрос?

| Вопрос пользователя | Где искать ответ (Приоритет) |
|---------------------|------------------------------|
| *"Какая у меня группа крови?"* | 1. `KNOWLEDGE_BASE.md` (раздел "Общая информация") |
| *"Как изменился вес за год?"* | 1. Таблица `weights` (БД).<br>2. `HEALTH.md` (раздел "Физические параметры"). |
| *"Какой у меня висцеральный жир?"* | 1. Таблица `weights` (поле `visceral_fat`).<br>2. OCR последних скриншотов в чате. |
| *"Пил ли я вчера витамин Д?"* | 1. Таблица `supplements_log` (БД). |
| *"Результаты последнего анализа тестостерона"* | 1. `KNOWLEDGE_BASE.md` (раздел "Гормоны" или "Динамика показателей").<br>2. Папка `data/hormones/` (свежий PDF). |
| *"Мои цели по здоровью"* | 1. `HEALTH.md` (раздел "Цели и приоритеты"). |
