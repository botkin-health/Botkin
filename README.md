## 🔥 Логика Калорий и Дефицита

**Цель:** Поддержание дефицита 15% для снижения веса с учетом активности.

**Формула:**
```
TDEE = BMR (пассивный метаболизм) + Активные калории (из Garmin)
Цель = TDEE × 0.85 (дефицит 15%)
Остаток = Цель - Съедено
```

**Пример:**
- BMR: 1750 ккал (базовый метаболизм в покое)
- Активные: 450 ккал (ходьба 10000 шагов + тренировка)
- TDEE: 2200 ккал (общий расход)
- **Цель: 1870 ккал** (с дефицитом 15%)

**Адаптивность:** В день тренировки можно съесть больше, в день отдыха — меньше. Система автоматически подтягивает данные из Garmin Connect (шаги, активные калории) и рассчитывает персональную норму на каждый день.

**Подробнее:** [calories_logic_explained.md](.gemini/antigravity/brain/dbd3bc5c-217e-4af6-9679-e6de0775cc42/calories_logic_explained.md)

## 📊 Анализ здоровья (ОБЯЗАТЕЛЬНАЯ ПРОЦЕДУРА)

**Для пользователя и AI-ассистентов (Cursor, Windsurf, Antigravity и др.):**

При любом запросе на анализ тела, веса, питания, сна или тренировок **сначала** запускайте валидатор данных:

```bash
cd /Users/alexlyskovsky/HealthVault
python3 scripts/validate_health_data.py
```

**Зачем:** Валидатор проверяет полноту данных (Apple Health, Zepp Life, Garmin, SleepCycle, питание), выявляет неполные дни и даёт статус готовности к анализу. Без этой проверки легко сделать неверные выводы (например, «нет тренировок» при их наличии).

**Порядок действий:**
1. Запустить `validate_health_data.py` и посмотреть отчёт
2. При статусе «КРИТИЧНО» — сообщить пользователю о проблемах, не проводить полный анализ
3. При статусе «ХОРОШО» или «ПРЕДУПРЕЖДЕНИЕ» — продолжать анализ с учётом замечаний из отчёта

Подробнее: `docs/README_DATA_ANALYSIS.md`, `docs/ANALYSIS_ERRORS_PREVENTION.md`

## 🛠️ Разработка и Тестирование

В проекте настроена автоматическая система контроля качества с расширенным тестовым покрытием.

### 📊 Тестовое покрытие

**Всего тестов:** 69  
**Покрытие критичных функций:** ~80%  
**Все тесты проходят:** ✅ 100%

**Категории тестов:**
- Weekly Nutrition (17 тестов): сайра, порции, парсинг name/weight, цель по белку
- Nutrition Parsing (16): текстовый парсинг, БЖУ, граничные случаи, вес
- **Weekly Report (5)**: структура, сайра, белок, калории, пустые данные
- **Vitamins (3)**: базовое логирование, вариации, вчера
- Bot Handlers (4): voice, text, photo, витамины
- Supplements (3): сохранение, расписание
- Nutrition Logic (5): порции, дубликаты, regression bugs
- Garmin Sync (5): умная синхронизация, консистентность
- Smoke Tests (4): быстрая проверка работоспособности
- Другие (7): даты, модели, репозиторий

### Автоматический запуск
При запуске бота через команду `make run` автоматически выполняются проверки:
1. **Тесты (`pytest`)**: 69 тестов проверяют все критичные сценарии
2. **Типы (`mypy`)**: Проверяет код на ошибки типов данных

Если проверки не проходят, бот **не запустится**. Это защищает от случайных поломок.

### Ручные команды

```bash
# Запустить все тесты
make test

# Запустить конкретный набор
pytest tests/test_weekly_nutrition.py -v   # Тесты питания
pytest tests/test_garmin_smart_sync.py -v  # Тесты Garmin
pytest tests/smoke_test.py -v              # Быстрая проверка

# С покрытием кода
pytest tests/ --cov=core --cov=database

# Только проверка типов
make check-types

# Установить зависимости для разработки
make install
```

> **Для разработчиков (Cursor/Windsurf)**: Перед тем как считать задачу выполненной, обязательно запускайте `make test`, чтобы убедиться, что вы ничего не сломали.

### 🐛 Regression Protection

Тесты защищают от повторения следующих багов:
- ❌ Сайра не засчитывалась как жирная рыба → ✅ 6 тестов
- ❌ Двойное умножение порций (bug с пиццей) → ✅ 1 тест
- ❌ Продукты из БД не парсились (name/weight) → ✅ 3 теста
- ❌ Неправильная цель по белку (177г вместо 150г) → ✅ 2 теста
- ❌ Garmin синхронизировал все дни заново → ✅ 3 теста

## 💾 Бэкапы проекта

Перед передачей проекта другой системе или перед крупными изменениями рекомендуется создать бэкап.

### Создание бэкапа:

```bash
cd HealthVault
./scripts/create_backup.sh
```

Бэкап сохраняется в `~/backups/HealthVault_backup_YYYY-MM-DD_HH-MM-SS.tar.gz`

**Что включается в бэкап:**
- ✅ Исходный код (telegram-bot/, scripts/)
- ✅ Базы знаний (knowledge_base.json, nutrition_log.json, workouts_database.json)
- ✅ Документация (все .md файлы)
- ✅ Конфигурация (.gitignore, requirements.txt)

**Что НЕ включается:**
- ❌ Большие файлы данных (PDF анализов, фото, данные Garmin)
- ❌ Временные файлы (venv/, __pycache__/, logs/)
- ❌ Секреты (.env, API ключи)

### Восстановление бэкапа:

См. подробную инструкцию в **`RESTORE_BACKUP.md`**

**Кратко:**
1. Найти нужный бэкап: `ls -lh ~/backups/HealthVault_backup_*.tar.gz`
2. Распаковать: `tar -xzf ~/backups/HealthVault_backup_YYYY-MM-DD_HH-MM-SS.tar.gz`
3. Восстановить файлы (см. `RESTORE_BACKUP.md` для деталей)

## 📚 Справочник данных (Data Guide)

Где искать ответы на вопросы о здоровье, анализах и метриках.

### 1. Накапливаемые данные (Сервер/Бот)
Эти данные приходят через Telegram-бот и сохраняются в SQL базе данных (PostgreSQL).

| Тип данных | Таблица БД | Описание |
|------------|------------|----------|
| **Вес и состав тела** | `weights` | Данные с умных весов (скриншоты, Zepp Life) и Apple Health.<br>✅ Вес, % жира, **висцеральный жир**, мышцы.<br>❌ Apple Health часто не хранит висцеральный жир, он берется из OCR скриншотов. |
| **Питание** | `nutrition_log` | Ежедневные записи еды + фото блюд. Подсчет КБЖУ. |
| **Витамины** | `supplements_log` | Лог приема добавок (фактическое время приема). |
| **Активность** | `activity_log` | Шаги, калории, сон.<br>Синхронизируется с Garmin (через скрипт) и Apple Health. |

### 2. База знаний (Файлы / HealthVault)
Статичные данные, документы и экспорты. Лежат в папке `HealthVault/data/`.

| Папка / Файл | Содержимое |
|--------------|------------|
| `KNOWLEDGE_BASE.md` | **ГЛАВНЫЙ ИНДЕКС**. Сводная таблица всех анализов с 2009 года. Группа крови, динамика гормонов, витаминов. |
| `HEALTH.md` | **Профиль здоровья**. Текущие цели, привычки, диагнозы, "ground truth" (текущий вес, возраст). |
| `data/blood-tests/` | PDF файлы анализов крови (исходники). |
| `data/medical-records/` | Заключения врачей, ЭКГ, результаты обследований. |
| `data/apple-health/` | Экспорты XML из Apple Health (исторические данные). |
| `data/garmin/` | Выгрузки из Garmin Connect (пульс, тренировки, сон). |
| `data/workouts_database.json` | История тренировок (Кроссфит, Бег). |

### 3. Где искать ответ на вопрос?

| Вопрос пользователя | Где искать ответ (Приоритет) |

| *"Какой у меня висцеральный жир?"* | 1. Таблица `weights` (поле `visceral_fat`).<br>2. OCR последних скриншотов в чате. |

| *"Результаты последнего анализа тестостерона"* | 1. `KNOWLEDGE_BASE.md` (раздел "Гормоны" или "Динамика показателей").<br>2. Папка `data/hormones/` (свежий PDF). |
