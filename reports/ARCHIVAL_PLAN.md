# План архивации legacy данных

**Дата:** 2026-02-01

## Критерии архивации

**Архивировать (переместить в `archive/legacy/`):**
- Файлы **до 2026-01-06** (первые 3 недели января)
- Данные уже в Postgres ✅
- Не используются ботом активно

**НЕ архивировать (оставить в `data/`):**
- Файлы **с 2026-01-06** и новее
- Активно используются в анализе
- Могут понадобиться для сравнения

## Команды

```bash
# Создать архив
mkdir -p archive/legacy/weights

# Переместить старые веса (до 2026-01-06)
git mv data/weights/2025-*.json archive/legacy/weights/
git mv data/weights/2026-01-05.json archive/legacy/weights/

# Проверка
ls archive/legacy/weights/
```

## График удаления

**После архивации:**
- ✅ Сразу: переместить в `archive/legacy/`
- ⏳ **Через 2-4 недели:** Если не было проблем с БД → удалить из archive
- 🔄 Альтернатива: Упаковать в `.tar.gz` и хранить минимально

**Откат:** Если что-то пошло не так → восстановить из archive.
