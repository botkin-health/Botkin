- Выделить общую бизнес-логику в отдельную библиотеку `core`.
- Очистить структуру проекта, отделив код бота от данных и скриптов.
- Исправить проблемы с путями и импортами.
- Обеспечить надежность работы после изменений.

## 2. Выполненные изменения

### Структура проекта
Создана новая директория `core/`, в которую перенесена вся бизнес-логика из `telegram-bot/services/`. Теперь структура выглядит так:

```text
HealthVault/
├── core/                 # [NEW] Общая библиотека
│   ├── nutrition.py      # Логика работы с питанием
│   ├── supplements.py    # Логика добавок (ex. supplement_service.py)
│   ├── storage.py        # Работа с файловой системой
│   ├── voice_service.py  # Распознавание голоса
│   └── ...
├── data/                 # [MOVED] Все данные проекта
│   ├── products.json     # База продуктов
│   ├── supplements.json  # База добавок
│   └── logs/             # Логи (питание, добавки)
├── telegram-bot/
│   ├── bot.py           # Точка входа
│   ├── handlers/        # Обработчики Telegram (импортируют core)
│   └── services/        # Только специфичные для бота сервисы (state.py)
└── scripts/             # Скрипты (теперь используют core)
```

### Перемещение данных
- Файлы `products.json` и `supplements.json` перенесены из `telegram-bot/data/` в `HealthVault/data/`.
- Логи добавок теперь сохраняются в `data/logs/supplements/`.
- Медиа-файлы консолидированы в `data/media/`.

### Исправление кода
- Обновлены все импорты в `handlers/` и `scripts/` на использование `core`.
- В `bot.py` добавлен путь к корню проекта в `sys.path`.
- Исправлен баг с распознаванием "псилиума" (добавлены ключевые слова).
