## 🛠 Технический долг

- [ ] **Круговой импорт** `core/nutrition.py` ↔ `core/llm_food_processor.py`: `nutrition.py` импортирует `process_llm_food_data` на строке ~389 для re-export. Это circular dependency — при прямом импорте `llm_food_processor` (например, в тестах в изоляции) падает с `ImportError`. Временный workaround в тестах: `import core.nutrition` перед `from core.llm_food_processor import ...`. Правильный фикс: убрать re-export из `nutrition.py`, или перенести shared-функции в третий модуль.
- [ ] **Реорганизация `scripts/`**: 49 файлов в плоском списке. Разбить на `scripts/migrations/`, `scripts/backfill/`, `scripts/analysis/`, `scripts/archive/` (мёртвые скрипты). Не удалять — архивировать.
- [ ] **Реорганизация `core/`**: 20+ модулей в одной папке. Сгруппировать в подпакеты: `core/routing/`, `core/food/`, `core/health/`, `core/integrations/`. Сделать только если навигация реально мешает.
- [ ] **Удалить `database/repository.py`**: помечен как deprecated (legacy psycopg2). Проверить что нигде не используется и удалить.

## 💡 Идеи для улучшений (оценены, решили пока не делать)

> **Контекст**: стоимость API ~$8.50/мес при текущей нагрузке (~12 сообщений/день). Оптимизации ради экономии не окупаются — дороже написать код.

- ❌ **Двухуровневый роутер** (local classifier → LLM только для неоднозначного): сэкономил бы ~$4/мес. Не окупается при текущем объёме. Пересмотреть если нагрузка вырастет в 10х.
- ❌ **Кэш LLM-ответов** (SHA256 по промпту, TTL 24ч): имеет смысл если одинаковые завтраки каждый день. Нужен Redis — overkill для личного бота.
- ❌ **Переезд с OpenAI на Claude**: цена та же, риск регрессий в поведении. Нет смысла без другой причины.
- 💤 **Vision → сразу результат без вопроса**: убрать лишний round-trip "опишите блюдо" — GPT-4o Vision сразу отдаёт результат с порогом уверенности. Улучшит UX фото. Сделать когда надоест описывать.
- 💤 **Few-shot примеры в промпте меню**: добавить 3-5 примеров разных форматов (PDF-меню, Яндекс.Еда, фото ресторана). Улучшит точность без смены модели. Малое усилие.
- 💤 **Автобэкап PostgreSQL**: `pg_dump` по cron или systemd timer на сервере. Важно для безопасности данных — сделать при следующем деплое.
- 💤 **Pre-commit хуки** (ruff + mypy + pytest smoke): добавить `.pre-commit-config.yaml`. Помогает при вайбкодинге — ловит ошибки до коммита.

## 👤 Личное Здоровье (Цели 2026)

### 🎯 Тело и Форма (Рост 170 см, Возраст 48 лет)
- **Вес**:
  - 📉 **Цель: 70-72 кг** (BMI ~24.5, "комфортный вес" для 48 лет).
  - *Расчет*: Идеальный по формулам ~66 кг, но корректируем на +4-6 кг за счет мышц.
- **Композиция**:
  - Жир: Снизить **< 20%** (текущий ~28%).
  - 🏋️‍♂️ **Мышцы**: Прирост мышечной массы (повышение BMR, поддержка суставов).
- **Висцеральный жир**:
  - 📉 **Цель: < 10** (Критично: в 48 лет висцеральный жир >10 резко повышает риск инфарктов).
  - *Стратегия*: Только дефицит калорий + кардио (этот жир уходит первым).

### 🩸 Биомаркеры и Риски
- **Холестерин**: 📉 Снизить **LDL < 3.0** (текущий ~3.79), **Общий < 5.0**.
- **Давление**: 📉 Стабилизировать **< 125/80**.
- **Инсулинорезистентность**: Снижение висцерального жира улучшит чувствительность к инсулину.
- **Анализы**: Регулярный чек-ап для контроля динамики (липидограмма).

### 🔋 Образ Жизни и Энергия
- **Сон**:
  - 🛌 **7+ часов** регулярно. Высокое качество (Deep/REM).
  - 🌙 **Режим**: Отбой до 00:00.
  - 🔇 **Храп**: Снизить/Убрать (прямая связь с весом и алкоголем).
- **Garmin Body Battery**: 🔋 Улучшить средний показатель "заряда" (меньше стресса, лучше отдых).
- **Питание**:
  - **Чистое питание**: Без срывов переедания.
  - **Белок**: > 150г (строительный материал для мышц).
  - **Алкоголь**: 📉 Минимизировать. Если есть — мало и без закусок (переедания).
- **Восстановление**: 🧖‍♂️ Баня/Массаж (1 раз в неделю).
