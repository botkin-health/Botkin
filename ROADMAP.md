## 🎯 Приоритетные планы

### 1. Multi-User Support 👥 **[ВЫСОКИЙ ПРИОРИТЕТ]**

> **Цель:** Разрешить нескольким пользователям (ты + жена) использовать бота

**Что нужно:**
- [ ] Авторизация пользователей (whitelist Telegram IDs)
- [ ] Привязка данных к `telegram_id` (уже реализовано в БД!)
- [ ] Раздельное хранение данных пользователей
- [ ] Персонализация целей (белок, калории, вес)
- [ ] Админ-панель для управления пользователями

**Текущий статус:**
- 🟡 **Частично готово:** БД уже поддерживает multi-user (таблица `users`, поле `telegram_id`)
- 🟡 **Нужна доработка:** Бот сейчас hardcoded на `USER_ID=895655`

**Оценка времени:** 4-6 часов
**Сложность:** Средняя

---

### 2. Apple Health Export Integration 📱 **[СРЕДНИЙ ПРИОРИТЕТ]**

> **Цель:** Автоматизировать импорт данных из Apple Health

**Что есть сейчас:**
- ✅ Скрипты для парсинга Apple Health XML экспортов
- ✅ Исторические данные с 2015 года
- ⚠️ Только ручной импорт (экспорт → скрипт → БД)

**Что нужно:**
- [ ] Telegram команда для загрузки Apple Health XML
- [ ] Автоматический парсинг и импорт
- [ ] Интеграция данных:
  - [ ] Вес и состав тела
  - [ ] Активность (шаги, калории)
  - [ ] Сон
  - [ ] Тренировки
- [ ] Дедупликация с Garmin данными

**Оценка времени:** 6-8 часов
**Сложность:** Средняя-высокая

---

### 3. Vision API Caching 💰 **[НИЗКИЙ ПРИОРИТЕТ]**

> **Цель:** Снизить расходы на OpenAI Vision API

**Проблема:**
- Каждое фото еды → новый API call
- Повторная обработка одинаковых фото

**Решение:**
- [ ] Кеширование результатов Vision API (по hash фото)
- [ ] Локальное хранилище кеша (SQLite или Redis)
- [ ] TTL для устаревания кеша
- [ ] Админ-команды для управления кешем

**Оценка времени:** 3-4 часа
**Сложность:** Низкая-средняя
