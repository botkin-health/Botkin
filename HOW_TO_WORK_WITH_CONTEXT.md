# Как работать с контекстом в HealthVault

## 📋 Ответы на ваши вопросы

### 1. Почему использовался старый метод (pdftotext) вместо Google Vision API?

**Причина:** При первой попытке не хватало библиотеки `pypdfium2` в venv. Для быстрого решения использовал `pdftotext`, который уже был в системе.

**Исправлено:** 
- ✅ Установлены `pypdfium2` и `Pillow` в venv
- ✅ Теперь можно использовать Google Vision API
- ✅ Создан скрипт `extract_health_data_chatgpt.py` для ChatGPT Vision

### 2. Стоит ли переходить в другой чат ("бот", "сон")?

**Короткий ответ:** Нет, не обязательно. База знаний на диске - это источник истины.

**Подробнее:**

#### ✅ Используйте текущий чат для:
- Обработки новых анализов
- Обновления `knowledge_base.json`
- Работы с файлами на диске
- Создания аналитических отчетов

#### 🔄 Используйте специализированные чаты для:
- **"бот"** - разработка телеграм-бота, специфичные для бота задачи
- **"сон"** - анализ данных сна, специфичные для сна задачи

#### ⚠️ Важно:
База знаний (`knowledge_base.json`, `nutrition_log.json` и т.д.) хранится **на диске и в GitHub**, а не в контексте чата. Поэтому:
- Можно работать в любом чате
- Всегда читаем данные из файлов
- Контекст чата используется только для обсуждения

## 🎯 Как корректировать питание с учетом анализов

### Шаг 1: Читаем данные из базы знаний

```python
# В любом чате можно сказать:
# "Прочитай последние анализы из knowledge_base.json"
# "Сравни с летними анализами 2024-07-12"
```

**Что происходит:**
1. Я читаю `knowledge_base.json` с диска
2. Нахожу последние анализы
3. Извлекаю нужные показатели

### Шаг 2: Генерируем рекомендации

```python
# Запускаем скрипт:
python data/analysis/nutrition_recommendations_template.py
```

**Что делает скрипт:**
1. Читает `knowledge_base.json`
2. Анализирует показатели (холестерин, витамины и т.д.)
3. Генерирует рекомендации по питанию
4. Сохраняет в `data/analysis/nutrition_recommendations.json`

### Шаг 3: Применяем рекомендации

```python
# В чате можно сказать:
# "Прочитай рекомендации из data/analysis/nutrition_recommendations.json"
# "Обнови мой план питания с учетом этих рекомендаций"
```

## 📊 Архитектура: База знаний vs Контекст чата

### ✅ База знаний на диске (источник истины):

```
HealthVault/
├── knowledge_base.json          # Все анализы, история
├── data/
│   ├── nutrition/
│   │   └── nutrition_log.json  # Лог питания
│   ├── analysis/               # Аналитические отчеты
│   └── blood-tests/            # PDF файлы анализов
└── ARCHITECTURE.md              # Этот документ
```

**Эти файлы:**
- ✅ Хранятся на диске
- ✅ Версионируются в Git
- ✅ Являются источником истины
- ✅ Доступны из любого чата

### ❌ Контекст чата (временная память):

**Контекст чата:**
- ⚠️ Временная память для текущей сессии
- ⚠️ Может быть потерян при перезапуске
- ⚠️ Не синхронизируется между чатами
- ✅ Используется для обсуждения и помощи

## 🎓 Ключевые принципы

### ✅ Делай так:

1. **Всегда читай из файлов:**
   ```python
   # В коде или в чате:
   "Прочитай из knowledge_base.json"
   "Загрузи данные из nutrition_log.json"
   ```

2. **Обновляй файлы:**
   ```python
   # При изменении данных:
   "Обнови knowledge_base.json"
   "Сохрани рекомендации в data/analysis/"
   ```

3. **Используй контекст для обсуждения:**
   ```
   "Объясни, что означают эти показатели"
   "Как интерпретировать результаты?"
   ```

### ❌ Не делай так:

1. **Не полагайся на память чата:**
   ```
   ❌ "Помни, у меня холестерин 5.66"
   ✅ "Прочитай из knowledge_base.json"
   ```

2. **Не храни данные в контексте:**
   ```
   ❌ "Сохрани это в памяти чата"
   ✅ "Сохрани в knowledge_base.json"
   ```

3. **Не дублируй данные:**
   ```
   ❌ Хранить одни и те же данные в разных чатах
   ✅ Использовать общую базу знаний на диске
   ```

## 📝 Резюме

### 🔑 Главное правило:

> **База знаний на диске (и в GitHub) - это источник истины.  
> Контекст чата - это инструмент для работы с этой базой знаний.**

### 🎯 Что это значит на практике:

1. **Можно работать в любом чате** - данные всегда на диске
2. **Всегда читаем из файлов** - не полагаемся на память чата
3. **Обновляем файлы** - сохраняем изменения на диск
4. **Коммитим в Git** - версионируем изменения
