## 2026-06-22 — #191 #192

**CGM sync: persisted backoff + Trend._missing_ патч**

- `scripts/import/librelinkup.py`: backoff на логин теперь сохраняется на диск
  (`data/cache/llu_backoff.json`). Раньше `_login_blocked_until` жил только в памяти
  процесса → каждый `/sync` в новом subprocess сбрасывал его в 0 → повторные попытки
  аутентификации продлевали Cloudflare-бан (Retry-After 86400с).
- `scripts/import/librelinkup.py`: `Trend._missing_` патч — `pylibrelinkup.Trend`
  (IntEnum 1–5) падал с Pydantic ValidationError на `TrendArrow=0` (новый сенсор).
  Патч: неизвестные значения → STABLE.
- `scripts/import/librelinkup.py`: `client.graph()` обёрнут в try/except аналогично
  `client.latest()`.

Прецедент: CGM не синхронизировалась с 20.06; данные за 21.06 были в телефоне, но
отсутствовали в БД. Cloudflare-бан истекает ~22.06 21:00 UTC.

## 2026-06-21 — #190 / PR #194

**Общий гард: не заявлять что видишь данные без вызова инструмента**

- `core/agent_chat.py`: добавлена секция `# 🚫 НЕ ЗАЯВЛЯЙ ЧТО ВИДИШЬ ДАННЫЕ БЕЗ ВЫЗОВА ИНСТРУМЕНТА`
  в начало `UNIVERSAL_META_PROMPT` (приоритетнее всех остальных секций).
  Запрещено: утверждать «вижу / знаю / у тебя записано» без подтверждённого tool-результата;
  заявлять «данных нет» без вызова инструмента; описывать состояние пользователя по
  system prompt вместо свежих данных из БД.
  Прецедент: @maxurazaev 19–20.06.2026 — агент описывал данные без вызова инструментов.

## 2026-06-20 — #183 #184 #185 / PR #186 #187

**Prompt guards: fallback поиска еды, объяснение дашборда, meal_time по фразе**

- `core/agent_chat.py`: добавлен блок `# 📅 ПОИСК ЕДЫ: НЕ СДАВАЙСЯ ПОСЛЕ ПЕРВОГО ЗАПРОСА`
  — если `get_recent_meals(days=1)` вернул пустой список, агент обязан повторить
  с `days=3` перед выводом «данных нет». Прецедент: Кристина @ochkina пишет утром
  про вчерашнюю еду, агент говорил «ничего нет».
- `core/agent_chat.py`: добавлен блок `# 📊 ДАШБОРД — ЭТО СТАТИСТИКА, НЕ ЖИВОЙ ЛОГ`
  — объяснение что дашборд показывает 7–30-дневные тренды, а не сегодняшний лог;
  запрет отвечать «обновляется с задержкой».
- `telegram-bot/handlers/text.py`: функция `_temporal_phrase_to_time()` — при записи
  еды через текстовый роутер `meal_time` инферится из фразы («вечером» → 20:00,
  «утром» → 08:00, «в обед» → 13:00, «ночью» → 22:30); fallback — текущее время.
