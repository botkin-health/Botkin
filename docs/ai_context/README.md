## Принципы этих доков (best practices, проверенные опытом)

Эти правила взяты из:
- Anthropic guidance по `CLAUDE.md` (см. claude.ai/docs)
- Опыта команд использующих Claude Code, Cursor, Antigravity на больших проектах
- Личных fail'ов автора (см. `AI_CHANGELOG.md`)

### Правила для меня (Claude/Cursor) когда я работаю с кодом

1. **Single source of truth.** Один факт живёт в одном файле. Если ты обнаружил что схема описана в двух местах — починить, оставив одно. Никогда не дублируй ради удобства.

2. **«Почему», а не только «что».** Документируй design rationale, особенно когда вариантов решения было несколько. «Используем JSONB вместо нормальных таблиц для items потому что схемы блюд сильно разные и редко join'им» лучше чем просто «items: JSONB».

3. **Anti-patterns в явном виде.** Если есть граната-нога — нарисовать её рядом с правильным паттерном. Пример: «❌ `users.target_calories` нет — это поле живёт в `user_settings`. ✅ Читать через `get_user_settings(db, user_id)`».

4. **Real examples > prose.** Один работающий SQL-запрос или snippet ценнее 100 слов описания.

5. **File:line ссылки** где это применимо: `database/crud.py:172` лучше чем «функция get_nutrition_totals_by_date». Так Claude (и человек) сразу прыгают в нужное место.

6. **Документировать вместе с кодом.** Меняешь модель → меняешь `03_database_schema.md` в том же коммите. Стейл-доки хуже отсутствия доков потому что молча врут.

7. **Канонические команды.** «Чтобы посмотреть логи бота: `docker logs healthvault_bot --tail 50`». Дай конкретные команды а не «проверьте логи».

8. **Помечай актуальность.** Каждый файл начинается с `**Last verified:** YYYY-MM-DD`. Если читаешь и видишь что данные устарели на месяц — это сигнал не доверять без проверки.

### Правила для людей которые поддерживают доки

- При любом серьёзном изменении (новая таблица, удалённая команда, изменённый формат промпта) — обновить тематический файл, поставить новую дату.
- При большом редизайне — пересмотреть `README.md` (этот файл) на актуальность навигации.
- `AI_CHANGELOG.md` — append-only журнал, никогда не редактировать прошлое.
- Никогда не плодить «контекст-файл-2», «context_v3.md». Если файл устарел — переписать его, не создавать копию.

## Стек одной строкой

Python 3.11 · aiogram 3 · FastAPI · SQLAlchemy 2 (Mapped style) · PostgreSQL 16 · GPT-4o (OpenAI) + Gemini fallback · Telegram Bot API + Mini App · Docker Compose · Hetzner VPS.

## Что бот умеет (одной строкой)

Логирует еду (текст / фото / голос / мини-апп) и добавки → считает КБЖУ + клетчатку + дефицит относительно Garmin BMR → показывает дневник через `/day` и Telegram Mini App.

## Главное правило безопасности данных

**Все запросы к данным пользователя — с явным `WHERE user_id = X`.** В боте 3 пользователя (владелец 895655, user2 REDACTED_ID, user3 REDACTED_ID). Без фильтра суммируются все сразу.
