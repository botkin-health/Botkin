## Способы залогировать еду (эволюция)

С момента когда писалась эта дока появились новые пути ввода. Вот **актуальный список**:

| Способ | Когда использовать | Точность КБЖУ |
|---|---|---|
| **Текст в чат** | Самый частый. Александр пишет «ужин: курица 200г, рис 150г» | средняя — LLM угадывает по тексту |
| **Текст с явными КБЖУ** | Когда есть точные данные с упаковки/Я.Лавки | максимальная — бот берёт как есть |
| **Фото блюда** | Ресторан / тарелка | низкая — оценка по виду |
| **Фото упаковки** | Магазинный продукт | максимальная — читает этикетку «Масса нетто» |
| **Фото меню** | Ресторан с меню | средняя — берёт цены и КБЖУ из меню |
| **Голосовое** | На бегу | как у текста после транскрипции (AssemblyAI рус) |
| **Мини-апп** | Редактирование/добавление к существующему дню | максимальная — ручной ввод |

**Важно про дату:** бот понимает префикс «вчера», «позавчера», «19 апреля», «19-го апреля», «19-е апреля» (любые суффиксы). Можно писать в любом месте сообщения: и `19-е апреля: ...`, и `завтрак 19-е апреля: ...`. См. `handlers/text.py:extract_date_from_text`.

## Когда Claude помогает разобрать обед — что делать

1. **Уточнить порции.** Сколько именно граммов *он лично* съел (не общая готовка на семью).
2. **Если сложный рецепт** — посчитать пропорции (например, съел 1/2 блюда → делим всё пополам).
3. **Подготовить готовый текст** для бота в формате который не путает LLM (см. ниже).
4. **Указывать КБЖУ явно** где можно — бот возьмёт точно, а не угадает.
5. **Не спрашивать разрешения** на подготовку записи — делать сразу после обсуждения еды.

## Правильный формат записи (работает)

```
форель РыбаLove 250г: 520 ккал (Б:50 Ж:30 У:0)
спаржа замороженная 167г: 52 ккал (Б:5 Ж:0 У:7)
сливочное масло 5г: 37 ккал (Б:0 Ж:4 У:0)
оливковое масло 15г: 135 ккал (Б:0 Ж:13 У:0)
руккола 35г: 9 ккал (Б:1 Ж:0 У:1)
```

**Ключевые правила:**
1. Один ингредиент = одна строка
2. Всегда граммы (или мл для жидкостей)
3. КБЖУ в формате `XXг: YYY ккал (Б:X Ж:X У:X)` — бот распознаёт точно
4. Столовые ложки → граммы: 1 ст.л. масла = 15г, 1 ч.л. = 5г
5. **Половина блюда = половина ингредиентов** — считай сам перед отправкой

## Что бот теперь делает автоматически (помнить)

Эти фичи появились после марта 2026 — старые версии этой доки про них не знают:

### ✅ Автоматическое определение веса с упаковки

Если на фото видна «Масса нетто» / «Net weight» — бот её прочитает и использует. **Не надо уточнять «50г Bombbar» если фото есть.**

### ✅ Fiber backfill

Бот теперь автоматически добавляет клетчатку для каждого блюда:
- Если LLM вернул `fiber` — берётся как есть
- Иначе — `fiber_table.py` оценивает по названию (овощи ~2г/100г, бобовые 6-8г, цельнозерновые 3-10г, мясо/молочка/масло = 0)
- Для составных блюд (уха, борщ, пирог, сочник, печенье) — есть dish-level entries в той же таблице

⚠️ Для блюд **с null весом** (когда LLM не вернул `weight_grams`) клетчатка остаётся 0. Если такое случилось — указать вес явно при логировании.

### ✅ Алкоголь распознаётся отдельно

Бот считает напитки в `drinks` поле (стандартные дозы алкоголя). Бокал вина = 1.5 дозы, рюмка водки = 1, кружка пива 500мл = 2.

### ✅ Мини-апп для правок

После сохранения можно открыть мини-апп (`/start` → кнопка), увидеть весь день, ткнуть в любой item чтобы поменять вес. КБЖУ пересчитаются пропорционально.

### ✅ Daily-log добавок (с тапом)

Таб «💊 Добавки» в мини-аппе показывает план + чекмарки. Тап → залогировано с текущим временем. Без необходимости писать «принял Витамин Д» в чат.

## Типичные ошибки GPT-4o (избегать)

| Ошибка | Почему | Как исправить |
|---|---|---|
| Занижает калории салата | Группирует ингредиенты, теряет масло | Разбить на отдельные строки |
| Ошибается с форелью/лососем | Берёт среднее значение, не конкретный продукт | Указать КБЖУ явно |
| Сливочное масло: берёт 1 ст.л. = 12-13г | Путает чайную и столовую | Писать в граммах: «сливочное масло 5г» |
| Спаржа: считает ~60 ккал/100г | Ориентируется на свежую, не замороженную | Указать явно: 31 ккал/100г |
| Возвращает `weight_grams: null` | Не уверен в порции для готового блюда | Указать вес или принять стандартный 100г (фикс 2026-04-20) |

## Как помогать с анализом еды (не только запись)

Когда Александр спрашивает «сколько я съел за неделю белка» или «как у меня с клетчаткой» — путь:

1. **НЕ читать локальные JSON-файлы** — питание живёт в PostgreSQL.
2. **SQL probe** через SSH (см. `02_data_sources.md` шпаргалка).
3. **Помнить про read-time fiber enrichment:** SQL `SUM((totals->>'fiber')::numeric)` даст устаревшие значения. Если нужна точная клетчатка — либо через `/api/day` endpoint (он enriches), либо в SQL с COALESCE и собственным enrichment.

**Главная ошибка:** не путать «факт» (`supplements_log`, что реально принял) и «план» (`user_settings.supplements`, что планировал). Чек-лист добавок на дне = пересечение.
