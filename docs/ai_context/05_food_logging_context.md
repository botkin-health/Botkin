## Как работает бот — формат входного сообщения

### ✅ Правильный формат (работает)

Каждый ингредиент — на **отдельной строке**. Граммы обязательны.
Если указать КБЖУ явно — бот возьмёт их точно, а не угадывает.

```
форель РыбаLove 250г: 520 ккал (Б:50 Ж:30 У:0)
спаржа замороженная 167г: 52 ккал (Б:5 Ж:0 У:7)
сливочное масло 5г: 37 ккал (Б:0 Ж:4 У:0)
оливковое масло 15г: 135 ккал (Б:0 Ж:13 У:0)
руккола 35г: 9 ккал (Б:1 Ж:0 У:1)
помидоры черри 60г: 11 ккал (Б:1 Ж:0 У:2)
```

### ❌ Неправильный формат (бот ошибается)

```
Обед: брускетта с лосем (чиабатта 50г, сыр 30г, сёмга 25г),
салат (руккола 35г, черри 60г, масло 2 ст.л.)
```
Проблема: GPT-4o группирует блюда и пересчитывает КБЖУ неверно, особенно по калориям.

---

## Ключевые правила форматирования

1. **Один ингредиент = одна строка** (никаких запятых внутри блюда)
2. **Всегда указывай граммы** — бот их берёт как есть
3. **Указывай КБЖУ явно** если знаешь реальные значения с упаковки/сайта
4. **Столовые ложки → граммы**: 1 ст.л. масла = 15г; 1 ч.л. = 5г
5. **Половина блюда = половина ингредиентов** — считай сам перед отправкой
6. **Формат КБЖУ**: `XXг: YYY ккал (Б:X Ж:X У:X)` — бот понимает это точно

## Типичные ошибки GPT-4o в боте (избегай)

| Ошибка | Почему | Как исправить |
|---|---|---|
| Занижает калории салата | Группирует все ингредиенты в один элемент, теряет масло | Разбить на отдельные строки |
| Ошибается с форелью/лососем | Берёт среднее значение, не конкретный продукт | Указать КБЖУ явно |
| Сливочное масло: берёт 1 ст.л. = 12–13г | Путает чайную и столовую ложку | Писать в граммах: "сливочное масло 5г" |
| Спаржа: считает ~60 ккал/100г | Ориентируется на свежую, не замороженную | Указать явно: 31 ккал/100г |
