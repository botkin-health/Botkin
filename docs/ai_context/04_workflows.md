# Инструкции (Workflows) для ИИ

Этот файл содержит стандартные операционные процедуры (SOP), которым должны следовать ИИ-ассистенты (Cursor, Claude, Antigravity) при выполнении типичных задач в проекте HealthVault.

## 1. Добавление новой интеграции (Нового источника данных)
Если пользователя просит "Добавь интеграцию с платформой X":

1. Создайте скрипт `scripts/import_X.py`.
2. Если для интеграции требуются ключи/пароли: 
   - Добавьте их шаблон в `.env.example`.
   - Прочитайте их в скрипте через `os.getenv()`. Не хардкодьте!
3. Формат сохранения должен быть единообразным:
   - Если данные нужны боту "прямо сейчас" — пишите в Postgres напрямую через `database/crud.py`.
   - Если это сырая выгрузка или аналитика — сохраняйте в `data/source_name/log_name.json` (создав папку с помощью `os.makedirs(..., exist_ok=True)`).
4. Обязательно добавьте этот источник в `02_data_sources.md`.
5. Обязательно запишите задачу в `AI_CHANGELOG.md`.

## 2. Изменение/Обучение LLM промптов
Telegram-бот использует LLM для парсинга еды и витаминов (`core/llm_router.py`). 
Если нужно "обучить" бота новому продукту (например, бот путает Омега-3 и рыбу):
1. Не меняйте логику `llm_router.py`. Изменяйте **системный промпт**, который находится либо в `core/llm_router.py`, либо в отдельном текстовом файле (если он уже вынесен).
2. Обязательно проверьте, как бот распарсит проблемный текст локально, прежде чем отправлять.
3. Обязательно запишите изменения промпта в `AI_CHANGELOG.md`.

## 3. Внесение изменений в БД (Schema Migration)
При добавлении новых колонок в SQLAlchemy:
1. Добавьте колонку в `database/models.py`.
2. В проекте (пока) может не быть Alembic. Если его нет — создайте SQL-скрипт в `database/migrations/` (или вызовите `ALTER TABLE` через `psql` напрямую с согласия пользователя).
3. Обновите `03_database_schema.md`.
4. Обновите функции в `database/crud.py` для корректного чтения/записи нового поля.

## 4. Запись в AI_CHANGELOG.md
Это ВАЖНЕЙШЕЕ правило. Всякий раз, когда вы (Cursor, Claude или любой другой ИИ) завершаете логическую задачу (Feature, Bugfix, Refactoring), вы обязаны открыть `docs/ai_context/AI_CHANGELOG.md` и добавить туда краткую запись.
**Формат:** `[YYYY-MM-DD] Описание (Какие файлы затронуты) - Автор (Ваше Имя)`.
*Пример:* `[2026-03-01] Добавлен импорт Screen Time (scripts/import_screentime.py) - Antigravity`.
