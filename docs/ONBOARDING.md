## TL;DR (1 минута) ⭐

**Проблема.** В 48 лет хочется жить хорошо и долго. Есть 15+ источников данных о здоровье: Garmin, iPhone, Apple Health, умные весы, домашняя метеостанция, ежеквартальные анализы крови, МРТ/УЗИ архив за 15 лет, субъективные ощущения. Все они размазаны по разным приложениям, экспортам и PDF-кам. Ни одно коммерческое решение не сводит это вместе — у каждого своя песочница.

**Решение.**
- **HealthVault** — методология: как собрать в одно место, как структурировать знания о себе и семье, какие корреляции искать, как вести диалог с AI про своё здоровье. Это skills для Claude Code, схема `knowledge_base.json`, промпты, подходы. Код тут минимальный — это **фреймворк для того, кто готов вести дисциплинированную работу со своим здоровьем**.
- **NutriLogBot** — практический инструмент в составе HealthVault: Telegram-бот, куда фотографируешь еду / диктуешь голосом / пишешь текстом → LLM распознаёт КБЖУ → всё пишется в Postgres, показывается в мини-приложении внутри Telegram. На нём удобно держать пищевой дневник семьи (сейчас 3 пользователя). Остальные метрики (сон, пульс, вес, АД, шаги) автоматически тянутся с устройств.

**Что сейчас работает:** бот у меня в проде с января 2026. Жена (Ника) пользуется с марта. 18 потоков данных синкаются регулярно. База знаний — все мои анализы с 2014, все УЗИ/МРТ/ЭКГ, дневные метрики с 6 января 2026 (этот день — старт системного трекинга).

## Часть 1. HealthVault — методология

### Идея

HealthVault — не приложение. Это **договорённость с самим собой** о том, как вести здоровье как проект. Инструмент — Claude Code (или любой AI-агент с файловой системой). Хранилище — папка на Google Drive + Postgres для операционных данных.

**Главное: у каждого члена семьи — своя папка с `knowledge_base.json` и `PROFILE.md`**. Это «медицинская карта» в машиночитаемом виде. Когда возникает вопрос «какой у мамы был уровень витамина D в 2022 году» — Claude читает её JSON, находит запись, даёт ответ со ссылкой на PDF. Когда нужно обсудить липидный профиль с терапевтом — генерирует сводку из 8 замеров за 6 лет.

### Структура семейного хранилища

```
Google Drive / HealthVault/
├── Александр Лысковский — Здоровье/     # я (48 лет, Москва)
│   ├── knowledge_base.json              # 564 значения: анализы, УЗИ, ЭКГ, МРТ, генетика
│   ├── PROFILE.md                       # карта диагнозов и хронических состояний
│   ├── TODO.md                          # что сделать по здоровью (опционально)
│   └── <год>/<тип>_YYYY-MM-DD_<лаб>.pdf # архив PDF-анализов
├── Валерия Лысковская — Здоровье/       # мама (72, Новосибирск)
├── Олег Лысковский — Здоровье/          # старший сын (24, Новосибирск)
├── Игорь Лысковский — Здоровье/         # младший сын (21, Новосибирск)
└── Екатерина Лысковская — Здоровье/     # бывшая жена, мама Олега и Игоря
```

**Ключевые правила:**
1. НИКОГДА не путать данные разных людей — каждая папка изолирована.
2. `knowledge_base.json` — единственный источник истины. PDF — архив оригиналов для подтверждения.
3. У каждого человека разная модель участия:
   - Ника (жена) — пользователь NutriLogBot, но `knowledge_base.json` веду я, просматриваем вместе
   - Дети в Новосибирске — у каждого может быть свой Claude; базой пока веду я
   - Мама — за неё полностью я (она не владеет инструментами AI)
4. В `knowledge_base.json` — структурированные секции: `blood_tests`, `urine_tests`, `hormones`, `vitamins`, `genetics`, `ultrasound`, `mri`, `ecg`, `medical_records` (записи приёмов).

### Claude skills (под капотом HealthVault)

Это slash-команды, которые я добавил себе в `~/.claude/skills/`. Каждый skill — это просто папка с `SKILL.md` (markdown-инструкцией для Claude).

| Skill | Что делает |
|---|---|
| `/sync` | Тянет свежие данные из всех источников: Garmin, Zepp, Netatmo, погода, Postgres dump. Показывает таблицу актуальности. ~3 минуты работы. |
| `/dashboard` | Показывает сводку: какие потоки данных актуальны, где провалы, что надо обновить. |
| `/chart` | Генерирует 3-панельный PNG (вес / калории / тренировки) за всё время. |
| `/cleanup` | Убирает за собой: удаляет `__pycache__`, `.DS_Store`, коммитит, пушит, делает бэкап БД, ротирует старые бэкапы. |

**Эти skills — часть методологии.** Они решают три боли:
1. «Где я оставил данные» — всё лежит по предсказуемым путям.
2. «Когда последний раз синкал» — в начале любой сессии `/sync` всё проверит.
3. «Что мне делать, чтобы рабочее место не сгнило» — `/cleanup` по расписанию.

### Найденные за 2026 наблюдения (примеры работы системы)

- **Корреляция холестерина и стресса (ложная).** На 12 днях r=+0.34. После детренда (вычли общий тренд по времени) — r=0. Оказалось, оба параметра просто плавно менялись по времени — ложная корреляция трендов.
- **Сон и HRV.** За 100+ дней видна стабильная связь «поздний отход ко сну → HRV ниже на следующий день на 8–12 мс». Это уже используется — стараюсь ложиться до 23:30.
- **Вес и питание.** Точная связь «недельный дефицит калорий → −0.4 кг за неделю» подтверждена на 3 месяцах. Сбросил 8 кг за 14 недель.
- **Давление и CO₂ в спальне.** Netatmo показывает CO₂ ночью 800–1400 ppm. Утреннее АД выше при CO₂ >1200 на ~5 мм рт. ст. Проветриваем спальню перед сном.

Это то, что не находится без сведения всех данных в одно место.

## Часть 2. NutriLogBot — продукт ⭐

### Идея

MyFitnessPal / FatSecret / Lifesum умеют записывать еду, но:
- Ужасно с голосом («я съел 200 грамм плова и два мандарина»)
- Ужасно с фотографиями («вот тарелка, разберись сам»)
- Не сводят питание с другими метриками (вес, сон, активность)
- Не работают с российскими продуктами (например, «творожок Савушкин»)
- Платные

**Гипотеза:** если у тебя под рукой LLM, можно сделать бот, который принимает свободный формат (текст / голос / фото), надёжно распознаёт еду и ведёт дневник. Плюс интеграция с Garmin / Apple Health / умными весами — чтобы цели по калориям и белку считались из твоих реальных трат, а не «средний мужчина 48 лет».

### Что умеет сейчас

```
Пользователь пишет/диктует/фотографирует              → LLM
      "Съел 200г курицы с овощами"                      ↓
      <voice: 15 секунд>                           транскрибация
      <фото тарелки>                               распознавание еды
                                                         ↓
                                              КБЖУ + граммовки + продукты
                                                         ↓
                                           Postgres (nutrition_log)
                                                         ↓
                             Пользователь видит в мини-приложении:
                             - Сегодняшние калории / цель (слайдер −30%…+20%)
                             - БЖУ прогресс-бары
                             - Добавки (план vs принято)
                             - Вес, BMR, активность
                             - Погода, CO₂ дома, сон, HRV
```

### Команды бота

- `/start` — приветствие, создание юзера в БД
- `/day` — сводка за сегодня (ккал, БЖУ, добавки, вес, шаги)
- `/week` — сводка за неделю
- `/vitamins` — принял / не принял сегодня
- `/help` — справка
- служебные: `/setup`, `/activity`, `/burn`, `/status` и пр.

### Мини-приложение (Telegram WebApp)

Это одностраничное приложение на чистом HTML/CSS/JS внутри Telegram. Кнопка открывается из любого сообщения бота. Разделы:
- **Сегодня** — съеденное, прогресс, кнопки редактирования
- **Добавки** — расписание витаминов / adaptogens с чекбоксами
- **Настройки** — BMR, цель по калориям (слайдер), целевой вес, дедлайн
- **Графики** — вес, калории, давление, сон — за 1 / 4 / 12 / 52 недели

Мини-аппа — в одном файле `telegram-bot/webapp/index.html` (~120 КБ). Это сделано намеренно: один файл = деплой одним `scp`, никаких сборщиков, никаких npm. Вся логика — в ~3000 строк чистого JS.

### Архитектура

```

│  Telegram User  │ ←→ │   aiogram bot   │ ←→ │   PostgreSQL     │

└─────────────────┘    └────────┬────────┘    └────────┬─────────┘
                                │                      │
                                ↓                      ↑
                       ┌─────────────────┐    ┌──────────────────┐
                       │   FastAPI       │ ←→ │   Mini App       │
                       │  (apple_health, │    │  (webapp/*.html) │
                       │   nutrition_api,│    │  in Telegram     │
                       │   supplements_  │    │   WebView        │
                       │   api)          │    └──────────────────┘
                       └────────┬────────┘
                                │
                       ┌────────▼────────┐
                       │  External APIs  │
                       │  Garmin, OpenAI,│
                       │  Gemini, OCR,   │
```

### Цикл типичного дня

- **Утром** — Garmin автоматически синкает сон → открываю бот, набираю «завтрак 2 яйца / творог 200 / мёд 20 / кофе» → через 3 секунды приходит сохранение с разбивкой КБЖУ. Открываю мини-аппу — смотрю сколько осталось по ккал на день.
- **Днём** — фотографирую бизнес-ланч → LLM распознаёт блюда, граммовки, КБЖУ → подтверждаю.
- **Вечером** — замер АД (Omron) → запускаю Shortcut на iPhone → webhook записывает в Postgres. Взвешивание (Zepp) → автоматически в CSV → следующий `/sync` подтянет.

## Часть 3. Как это хостится и деплоится

**Сервер:** Hetzner, Ubuntu 22, IP `116.203.213.137` (`jarvis`).

**Compose-стек:**
- `healthvault_bot` — Python-контейнер с ботом + FastAPI
- `healthvault_postgres` — PostgreSQL 16 с volume для данных
- `caddy` — reverse-proxy с автоматическими SSL (Let's Encrypt) для домена `health.orangegate.cc`

**Деплой:** `git push origin main` → вручную `scp` изменённых файлов на сервер + `docker restart healthvault_bot`. CI/CD пока нет — скрипт `deploy.sh` обновляет всё. На каждый пуш запускается pre-commit локально (ruff, формат, запрет секретов).

**Бэкапы:** cron ежедневно в 04:17 UTC делает `pg_dump` → `data/backups/` на сервере. Ротация — 14 последних. Локальные бэкапы тянет `/cleanup` по запросу.

**Логи:** `docker logs healthvault_bot -f`. Структурное логирование через `logging`, уровень из `LOG_LEVEL` env.
