## Зачем

После «Server-side sync» (мая 17-19) мы перенесли все pull-источники на сервер. Но **биомаркеры (анализы крови)** остались завязаны на мак: источник истины — `~/FamilyHealth/Александр Лысковский — Здоровье/knowledge_base.json`, а derived-файл `biomarkers_895655.json` на сервере обновляется только через ручной `python3 scripts/generate_biomarkers_json.py --deploy`.

Это блокирует:
1. **Личный комфорт** — Claude Desktop (Александр в чате со мной) лезет в filesystem мака, не видит свежие данные на сервере, рассинхрон.
2. **Семью** — Андрей, Олег, Игорь, Ника, папа не могут добавлять анализы (у них нет FamilyHealth-папки и доступа к репо).
3. **Native iOS/Android в будущем** — приложение должно работать через API, а KB на маке не доступен снаружи.

## Архитектурный итог

```
Источник истины биомаркеров → PostgreSQL на сервере (новая таблица biomarkers)
                              ↑                            ↓
                         PDF/JPEG-парсер             agent_tools_api.py
                         в Telegram-боте              (JWT + RLS)
                         (Claude API)                       ↓
                              ↑                            ↓
                              ↑                  ┌──────┴──────┐
                              ↑                  ↓             ↓
                      Пользователи         botkin-mcp     Дашборд
                      кидают PDF в бота    (Claude Desktop)  HTML
```

- `knowledge_base.json` остаётся как **исторический бэкап** на маке для Александра, не как источник истины
- PDF/изображение приходит → парсится Claude API → структурированные данные в БД → оригинал удаляется
- Дашборд читает биомаркеры напрямую из БД (никаких derived JSON-ов)
- Claude Desktop ходит через MCP-сервер → тот же `agent_tools_api.py` → БД
