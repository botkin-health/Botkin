# ❓ FAQ — типичные вопросы

## Общее

### Это бесплатно?

Сам сервис — да. **Затраты** (для меня как owner): Hetzner-сервер ~10€/мес, OpenAI/Anthropic API ~$30–50/мес при 5 юзерах. Если станет 50+ пользователей и расходы вырастут — будем думать про BYOK (Bring Your Own Key — каждый платит за свои LLM-токены сам) или подписку.

### Это медицинский совет?

**Нет.** Это инструмент для дисциплинированного ведения дневника и обсуждения трендов с AI. AI хорошо суммирует, ищет корреляции, объясняет биомаркеры — но **не заменяет врача**. По всем важным решениям — к своему врачу.

### Кто за этим стоит?

Александр Лысковский, [@lyskovsky в Telegram](https://t.me/lyskovsky). Бывший CEO Alawar (геймдев), сейчас в DAO Tech и автор Botkin как личного проекта. Семья — часть пользовательской базы и первые бета-тестировщики.

### Можно ли работать с ботом без Apple iPhone?

Бот работает у любого Telegram-юзера. Без iPhone не будет автоматического синка Apple Health (часы/тонометр/весы) — но всё что вводишь вручную через бот будет работать. Если у тебя Garmin — есть отдельный коннектор (требует логин-пароль от Garmin Connect).

### Где код?

Открытый: [github.com/Lyskovsky/Botkin](https://github.com/Lyskovsky/Botkin).

## Telegram-бот

### Я попал на бота с украинским интерфейсом, это что?

Это **чужой бот-двойник «Nutrilog»**, не наш. Это нутрициологический бот на украинском, не имеет к нам отношения. Наш правильный бот — `@Botkin_md_bot`, в шапке чата у него написано **«Botkin»**. Прямая ссылка: [t.me/Botkin_md_bot](https://t.me/Botkin_md_bot).

### Можно ли отправить голосовое на 5 минут?

Whisper иногда обрывает после ~2 мин. Лучше режь на части по приёмам пищи.

### Бот неправильно распознал еду

Напиши вторым сообщением «нет, это было 250г а не 400» или «там был ещё творог 150г» — бот пересчитает. Или нажми кнопку «✏ Изменить» под распознанным.

### Бот не отвечает

- Проверь интернет
- Бот в Docker, иногда перезапускается — подожди минуту
- Если 5+ минут — напиши Александру [@lyskovsky](https://t.me/lyskovsky)

### Как удалить запись о еде

В Mini-app → Дневник → тап на еду → кнопка «Удалить». Или в боте: «удали последнее».

## Apple Health

### Health Auto Export ничего не шлёт

Проверь по порядку:
1. **Background App Refresh** для HAE включён в iOS Settings → General → Background App Refresh → HAE → ON
2. **Low Power Mode** выключен (он блокирует фоновое)
3. iPhone разблокировывался хотя бы раз за сутки
4. В HAE → Automations → твоя автоматизация enabled (зелёная)
5. Сделай Manual Export в HAE — если ручной работает, значит автоматика просто ждёт ночь

### iOS Shortcut вместо HAE — работает?

Да — это официальный **бесплатный путь** (endpoint v1, `/apple_health`): готовый iCloud-шаблон Shortcut описан в [гайде Apple Health](apple-health.md). Компромисс: Shortcuts менее надёжны — могут требовать ручного запуска и иногда падают. Если хочется «настроил и забыл» — HAE (v2 endpoint) работает в фоне автоматически, но стоит $24.99.

### Почему HAE платный?

Pro-версия с REST API стоит $24.99 lifetime — это разовая покупка. Бесплатная версия не умеет REST API (только iCloud экспорт). Без REST API нельзя автоматизировать. Однажды заплатил — на всю жизнь работает.

### У меня Android, что делать

Apple Health не работает на Android. Альтернативы:
- **Garmin Connect** — если у тебя Garmin часы, есть прямая интеграция (нужен логин/пароль от Garmin)
- **Google Fit / Health Connect** — пока не интегрировано, в плане на v1.1
- **Ручной ввод** — пиши в бот вес/АД текстом, через мини-апп

## Claude Desktop MCP

### Claude не видит Botkin tools

1. Конфиг сохранён? `claude_desktop_config.json` валидный JSON (запятые везде где надо)?
2. Полный Cmd+Q + перезапуск Claude (не просто закрытие окна)
3. Токен корректный? Проверь в боте `/mcp_token`

### Claude получает 401 при вызове tool

Токен устарел. В боте: `/mcp_token rotate` → скопируй новый → обнови в config → перезапусти Claude.

### Я хочу подключить и приватные локальные файлы

Тебе нужен второй MCP — `filesystem`. Документация: [modelcontextprotocol.io/docs/quickstart/user](https://modelcontextprotocol.io/docs/quickstart/user). Добавляешь в тот же `mcpServers` блок, указываешь пути к локальным папкам. Claude видит и сервер (через Botkin MCP) и локалку (через filesystem MCP) одновременно.

### Могу ли я случайно навредить своим данным через Claude

Теоретически — да, write-tools могут что-то логировать. Практически — Claude **спрашивает подтверждение** перед записью (если не отключено). Если боишься — попроси Claude использовать только read-tools («не вызывай log_*, только get_*»).

### Я хочу удалить всё что у меня в Botkin через Claude

Сейчас нет delete-tool в MCP (намеренно — чтобы случайно не снести историю). Если хочешь — через Александра или команду `/delete_my_account` в боте (когда будет реализовано, Sprint 5).

## Knowledge Base

### Что если я не хочу подключать Google Drive

Будь `kb_status='private'` или `'none'`. AI всё равно будет работать с тем что в БД (питание, вес, шаги). Анализы и заключения — не будут в дашборде/боте, но если ты подключил MCP в Claude Desktop — у тебя там через локальный filesystem всё видно.

### А что если у меня старые анализы на бумаге

Сфотографируй iPhone-камерой → получится JPEG → положи в свою KB-папку с правильным именем → попроси Claude (через MCP) распознать значения и добавить в `knowledge_base.json`. Sprint 6 — это будет автоматически через watcher.

### Сколько данных может хранить knowledge_base.json

Технически JSON может быть огромным, но для удобства парсинга/чтения LLM — желательно <500 KB на юзера. У Александра сейчас ~150 KB (4 года записей), у Олега ~50 KB. Если станет огромным — разнесём по подфайлам (`vitamin_d.json`, `psychiatry.json`, etc).

### Чувствительные данные в KB — что с ними

Если у тебя `kb_status='shared'` — все файлы в папке видят owner и AI. Если что-то слишком приватное — храни **локально** (не клади в shared папку) и работай через Claude Desktop с filesystem MCP.

## Безопасность

### Что если Александр (owner) решит подсмотреть мои данные

- Через `/admin/` — он видит **агрегаты** (количество записей, последняя активность) но не **детали** в текущем MVP. В Sprint 5 — добавим клик на юзера → детали (с твоего согласия)
- Через psql на сервере — может прочесть всю БД. Но это будет залогировано в `audit_log` (Sprint 6). Ты увидишь.
- Через твою Google Drive shared папку — он её видит изначально, ты сам её расшарил. Если не хочешь — поменяй статус на `private`

### Я случайно закоммитил `claude_desktop_config.json` с токеном в публичный git

1. **Немедленно** в боте: `/mcp_token rotate`
2. **Force-push** старого коммита из истории git (или удали ветку, или используй BFG Repo-Cleaner)
3. Старый токен утёк, но больше не работает

### Если меня взломают и угонят Telegram

Без доступа к твоему Telegram-аккаунту никто не получит твои токены (они выдаются только через бот в DM). Если взломали Telegram целиком — попроси Александра вручную через `/admin/` отозвать твои токены, восстанови Telegram, потом перевыпусти.

Да, через Claude Desktop MCP можешь сам выгрузить всё. Или попроси Александра — он сделает SQL-выгрузку в CSV/JSON.

## Технические

### Сервер где

Hetzner CCX13 в Берлине. Германия (GDPR-юрисдикция).

### Что если сервер упадёт

- Бэкапы Postgres ежедневные, восстановление в течение 1-2 часов
- Бот в Docker, рестартится автоматически если контейнер падает
- Disk usage сейчас 63% — есть запас
- Через `/admin/` Александр видит статус сервера в реальном времени

### Я хочу свой собственный self-hosted экземпляр

Открытый исходный код, можно. Клонируешь репо, ставишь Postgres + Docker + Caddy, регистрируешь своего бота через @BotFather, прописываешь токены в `.env`. Документация для разработчиков — в `docs/ai_context/`. Александр готов помочь если возьмёшься.

### Какие LLM-модели используются

- **Claude Sonnet 4** — для распознавания фото еды, conversational agent
- **OpenAI Whisper** — для расшифровки голосовых
- **Claude Sonnet 4 (через OpenAI compat)** — для парсинга текстовых запросов

Можно перейти на BYOK (свои API-ключи у каждого юзера) — план в `todo.md`, пока не реализовано.
