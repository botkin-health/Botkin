# 🔒 Безопасность и токены

> Полная карта: что видит owner проекта, что лежит локально только у тебя, какие у тебя токены, как их отозвать.

## Кто и что видит — большая таблица

| Источник данных | Owner (Alex) | AI юзера в боте | AI юзера в Claude Desktop |
|---|:---:|:---:|:---:|
| Питание из бота | ✅ через `/admin/` если family/early_user | ✅ | ✅ через MCP |
| Добавки и лекарства | ✅ | ✅ | ✅ |
| Вес, % жира, давление | ✅ | ✅ | ✅ |
| Apple Health / Garmin данные | ✅ | ✅ | ✅ |
| Shared Google Drive папка | ✅ если расшарено | ✅ | ✅ |
| Profile (имя, возраст, диагнозы) | ✅ | ✅ | ✅ |
| **Private папка на маке юзера** | ❌ **никогда** | ❌ | ✅ через локальный filesystem MCP |
| **КПТ-дневник / intimate notes** | ❌ **никогда** | ❌ | ✅ через локальный MCP |
| Общий longevity KB (в репо) | ✅ публично | ✅ | ✅ |
| Чужие данные (других юзеров) | ✅ admin может в БД, но логируется | ❌ | ❌ |

## Архитектурное правило приватности

**Сервер хранит только то, что юзер сам туда положил** (через бот, через Apple Health, через мини-апп).

**Приватное (КПТ-дневники, intimate health notes) живёт у юзера на маке** и подключается к AI только в его личной сессии Claude Desktop через локальный filesystem MCP. Сервер их **физически не видит** — там нечего шифровать, нечего утечь.

Это намеренно. Архитектурно проще, надёжнее, понятнее.

## Твои токены

У каждого юзера есть **до двух токенов** в системе:

### 1. `health_token` — для Apple Health Auto Export

**Назначение:** твой iPhone через приложение HAE шлёт данные на сервер. Сервер должен понять «это от юзера X».

**Что может:** только **писать конкретные метрики** (steps, weight, BP, heart_rate, sleep) в твою БД-запись.

**Что НЕ может:** читать чужие данные, удалять, менять профиль.

**Где взять:** в боте `/health_token` → копируешь в HAE app.

**Как обновить:** `/health_token rotate` в боте.

**Как отозвать:** `/health_token revoke` (потом перевыпустишь когда нужно).

### 2. `mcp_token` — для Claude Desktop MCP

**Назначение:** твой Claude Desktop через MCP читает твои данные с сервера.

**Что может:** читать **все** твои данные + писать новые записи (питание, добавки, давление, вес, обновлять профиль).

**Что НЕ может:** видеть чужие данные, делать admin-действия.

**Где взять:** в боте `/mcp_token` → копируешь в `claude_desktop_config.json`.

**Как обновить:** `/mcp_token rotate` в боте.

**Как отозвать:** `/mcp_token revoke`.

## Почему два токена, а не один

Разные «силы» нужны для разных задач:

| | HAE token | MCP token |
|---|---|---|
| Кому даём | Apple Health Auto Export | Claude Desktop |
| Где живёт | В настройках HAE app на iPhone | В config-файле Claude Desktop на маке |
| Что может | Писать узкий набор метрик | Читать всё + писать |
| Если утечёт | Кто-то может слать фейковые шаги/вес. Не катастрофа | Кто-то может прочитать всё. Серьёзнее |
| Ротация | Раз в год вряд ли понадобится | Чаще: например, при смене мака |

Раздельные ключи — **best practice безопасности**. Если один утечёт — потерян только один доступ, не всё.

## Как настроить и поменять токены

### Через Telegram-бот (повседневное)

```
/health_token         показать текущий
/health_token rotate  перевыпустить
/health_token revoke  отозвать

/mcp_token            показать или выдать
/mcp_token rotate     перевыпустить
/mcp_token revoke     отозвать
```

### Через Mini-app

Открой мини-апп → вкладка **Настройки** → секция **🔑 Токены**:
- Видишь статус каждого токена (выдан / не выдан, дата выпуска)
- Кнопки **[Показать]**, **[Перевыпустить]**, **[Отозвать]**

### Через admin-дашборд (только Александр)

В таблице юзеров колонка **🔑 Tokens** показывает статусы. Александр может **перевыпустить или отозвать твои токены** только в критических случаях:
- Ты потерял доступ к Telegram → тебе нужен новый токен но через бот не получишь → Александр поможет
- Подозрение на компрометацию → отзыв немедленно

Любое действие админа логируется (audit log).

## Что делать если токен утёк

**Симптомы:**
- Случайно закоммитил `claude_desktop_config.json` в публичный git
- Поделился скриншотом с видимым токеном
- Подозрение что кто-то лазает в твоих данных

**Действия:**

1. В боте: `/mcp_token rotate` (или `/health_token rotate` для HAE)
2. Старый токен сразу перестаёт работать
3. Новый записан в БД, его можно скопировать
4. Если был коммит в публичный git — **обязательно сделай force-push** старого коммита (rotate токена не делает старый коммит безопасным, токен виден в истории git)

## Как обеспечивается изоляция

Все API на сервере (Tools API для бота, MCP server, mini-app) при каждом запросе:

1. Берут токен из `Authorization: Bearer ...`
2. Lookup в `users` по этому токену → находят `user_id`
3. Все последующие SQL содержат `WHERE user_id = $user_id`
4. Возвращают только твои строки

Если попытаться запросить «дай мне данные юзера X» с твоим токеном — сервер вернёт **только твои**, потому что user_id привязан к токену, а не задаётся в payload.

**Sprint 6 (план):** добавим Postgres Row-Level Security (RLS) — дополнительный слой защиты на уровне БД, если в коде Tools API когда-то будет баг, RLS не пропустит.

## Audit log (план — Sprint 6)

Когда Александр (admin-роль) делает любой SELECT/UPDATE — это **логируется в `audit_log`** таблицу. Любой юзер сможет запросить «покажи мне что Александр делал с моими данными за последний месяц» — и увидит точные timestamps и запросы.

Пока (до Sprint 6) audit log нет — есть только обещание не делать.

## mcp_access_log (план — v1.1)

Логирование каждого вызова MCP с твоим токеном: когда, с какого IP, какой tool. Чтобы ты сам видел «мой Claude обращается / кто-то ещё с моим токеном лазает».

В MVP не делаем — добавим когда юзеров станет 10+.

## Бэкапы

Чтобы данные не потерять физически:

- Postgres дампится в `/opt/backups/` ежедневно (Sprint 6 — ставим автоматически)
- Можно сделать вручную через `/admin/` (есть кнопка «Сделать бэкап сейчас»)
- Хранение бэкапов на сервере, копия на Hetzner Storage Box — план в `todo.md`

## Если хочешь полностью удалить свои данные

Напиши в боте `/delete_my_account` (команда добавляем в Sprint 5) или просто скажи Александру. Действия:

1. Все твои записи в `nutrition_log`, `supplements_log`, `weights`, `activity_log`, `blood_pressure_logs` — удаляются
2. Запись в `users` — удаляется
3. Все твои токены — отзываются
4. Бэкапы остаются 30 дней (по политике хранения), потом тоже удаляются
5. Shared Google Drive папку (если была) — переносишь к себе сам
