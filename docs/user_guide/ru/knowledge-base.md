# 📂 Knowledge Base — твоя база знаний о здоровье

> Чтобы AI-помощник мог давать осмысленные ответы про твоё здоровье, ему нужны не только данные из бота, но и **медицинская история**: анализы крови, заключения врачей, УЗИ, МРТ, прививки. Это и есть твой Knowledge Base.

## Что такое Knowledge Base

Это **папка** где лежат:
- PDF / JPEG / DOCX оригиналы анализов и заключений
- `knowledge_base.json` — структурированная сводка (числовые значения, даты, файлы)
- `PROFILE.md` — живая карта здоровья (диагнозы, текущая терапия, риски)
- `TODO.md` — план обследований/анализов которые предстоит сделать
- `CLAUDE.md` — инструкция для AI-агента как читать эту папку

AI-агент при ответе на вопрос вроде «у меня ферритин 112 — это норма?» **читает твою KB**, видит динамику ферритина за 5 лет, контекст диагнозов, и отвечает с учётом всего этого. А не общими словами «норма у мужчин 20–250».

## Три статуса KB

В админ-настройках для каждого юзера выставлен один из трёх:

### 🟢 Shared

Папка лежит **в общей Google Drive** семьи Лысковских:
```
~/Library/CloudStorage/GoogleDrive-lyskovsky@gmail.com/Мой диск/FamilyHealth/{Имя} — Здоровье/
```

**Кто видит:** owner проекта (Александр) + AI-агент юзера.

**Когда подходит:** семья с прозрачными отношениями, либо просто человек который не имеет ничего против чтобы Александр видел его анализы (например, чтобы посоветовать что-то).

**Текущие shared-юзеры:** Александр (свои), Олег, Андрей.

### 🔒 Private

Папка с медданными лежит **только на маке юзера**, никогда не попадает на сервер.

**Кто видит:** только сам юзер + его Claude Desktop через локальный filesystem MCP (см. [MCP](./mcp-claude-desktop.md)).

**Кто не видит:** Александр, бот, другие юзеры, любой кто получит доступ к серверу.

**Когда подходит:** если ты предпочитаешь полную приватность медицинских данных, и при этом активно пользуешься Claude Desktop. Жертвуешь автоматическим биомаркер-блоком в дашборде, но получаешь полный приватный контекст для AI в Claude.

**Текущие private-юзеры:** Ника.

### ⚪ None

KB не подключен вообще. Только данные которые юзер сам пишет в бот / синкает через Apple Health.

**Когда подходит:** ты только начал, нет архива анализов, или принципиально не хочешь подключать.

## Структура shared-папки (пример: Олег)

```
FamilyHealth/Олег Лысковский — Здоровье/
├── CLAUDE.md                            ← инструкция для AI как читать папку
├── PROFILE.md                           ← диагнозы, терапия, ключевые показатели
├── TODO.md                              ← план анализов на ближайшие недели
├── knowledge_base.json                  ← структурированная база (источник истины)
├── knowledge_base_backup_*.json         ← бэкапы
├── blood_2026-04-13_unknown_comprehensive.pdf   ← оригинал анализа
├── blood_2025-09-19_kdl_comprehensive.pdf
├── ecg_2025-09-30_emias_ecg_1.pdf       ← ЭКГ
├── psych_exam_2025-09-26_unknown_general.pdf  ← заключение психолога
├── psychiatrist_2026-04-08_unknown_general.jpeg  ← прием психиатра
├── uzi_abdomen_2021-02-21_unknown_general.pdf   ← УЗИ
├── mri_brain_neck_2020-06-25_unknown_general.pdf  ← МРТ
└── ...(всего ~164 файла)
```

### Соглашение по именам файлов

```
{тип}_{ГГГГ-ММ-ДД}_{источник}_{описание}.{расширение}
```

Примеры:
- `blood_2026-04-13_kdl_general.pdf` — общий анализ крови из лаборатории KDL за 13.04.2026
- `urine_2025-09-15_invitro_general.pdf` — анализ мочи Инвитро
- `uzi_thyroid_2020-07-13_unknown_general.pdf` — УЗИ щитовидки

**Правила:**
- Только латиница, без кириллицы и пробелов
- Дата всегда вторая в формате ГГГГ-ММ-ДД
- Если источник неизвестен — `unknown`

## Как пополнять

### Самый простой путь — команда /doc в боте (для всех)

1. Напиши боту `/doc`
2. Пришли PDF, фото или скан анализа/заключения (до 20 МБ; .docx — пересохрани в PDF)
3. Бот покажет превью найденных показателей (дата, лаборатория, значения)
4. Нажми «Сохранить ✅» — файл ляжет в архив на сервере, данные — в секцию
   `documents[]` твоего KB; агент увидит их через `kb_value`
5. Можно слать несколько документов подряд; выход — /cancel

Если числовых значений в документе нет — бот предложит «Сохранить как архив»:
файл сохранится, агент будет знать, что документ есть.

⚠️ Структурированный перенос значений в `blood_tests` (для дашборда/PhenoAge)
пока остаётся отдельной операцией — см. ниже.

### Если ты shared:

1. Сфотографируй или скачай PDF из лаборатории
2. Положи в свою папку с правильным именем
3. AI-агент (когда watcher будет запущен — Sprint 6) автоматически прочитает и обновит `knowledge_base.json`

**Прямо сейчас (до Sprint 6):** скажи в чат с Claude (если у тебя MCP подключен) — он распознает значения и добавит в KB.

### Если ты private:

То же самое, но папка у тебя на маке. Используй свой Claude Desktop с filesystem MCP — он видит файлы локально, читает их, обновляет твой локальный `knowledge_base.json`.

## Что внутри knowledge_base.json

Главные разделы (на примере типового файла):

```jsonc
{
  "patient_info": {
    "name": "...",
    "birth_date": "...",
    "diagnoses": ["..."],
    "medications_current": {...}
  },
  "vitamin_d": [
    {"date": "2025-09-19", "value": 26.3, "status": "insufficiency", "file": "..."},
    {"date": "2026-04-13", "value": 26.3, ...}
  ],
  "ferritin": [{...}],
  "blood_tests": [
    {
      "date": "2026-04-13",
      "lab": "...",
      "values": {
        "Hb": 165, "RBC": 5.82, "PLT": 130,
        "ALT": 24.3, "vitamin_d": 26.3, ...
      },
      "flags": {"PLT": "LOW (...)"},
      "note": "..."
    }
  ],
  "ecg": [...],
  "ultrasound": [...],
  "mri": [...],
  "psychiatry": [...],
  "medical_records": [...],
  "vaccinations": {...}
}
```

Это **источник истины** — все числа должны быть здесь. Файлы PDF — это **архив оригиналов**, не источник правды.

## Что внутри PROFILE.md

Человекочитаемая карта здоровья. Главные секции:

- **📇 Паспорт** — ДР, пол, группа крови, семья
- **🩺 Диагнозы** — МКБ-10 с датами установки
- **💊 Текущая фармакотерапия** — препараты, дозы, что мониторить
- **🚨 Зоны риска** — 🔴 высокий приоритет / 🟡 среднее / 🟢 хорошее
- **📊 Полная картина последнего анализа** — структурированная сводка
- **📅 История подбора терапии** — какие схемы пробовали
- **📜 Краткая медистория** — хроники

Этот файл **для тебя самого, для AI, и для врача** — можно распечатать и отдать на приёме.

## Privacy для shared

Если ты shared — Александр имеет доступ к твоей папке через свой Google Drive. Это **по твоему согласию**. Он не использует это во вред — только чтобы помочь (если попросишь) или для проектных задач (отладить парсер на твоих данных, например).

Если в какой-то момент захочешь приватность — скажи. Поменяем статус на `private`, перенесём папку к тебе на мак.
