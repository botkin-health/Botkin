# 🔌 Botkin MCP — личный доступ к своим данным через Claude Desktop

> Для пользователей Botkin, у которых на компе есть Claude Desktop: подключи Botkin-коннектор,
> чтобы Claude видел твои серверные данные и мог обсуждать твоё здоровье в любой сессии.

## Что такое MCP

**MCP** (Model Context Protocol) — стандарт, который даёт AI-агенту (Claude Desktop) доступ к внешним данным через набор инструментов. **Botkin MCP** — наш коннектор, который предоставляет Claude твои данные с сервера Botkin:

- питание за любой период (КБЖУ, продукты, калории)
- вес, % жира, давление, биомаркеры
- тренировки и активность
- журнал добавок и лекарств

**Приватные файлы** (КПТ-дневник, сканы анализов которые ты не загружал в Botkin, файлы мед.приборов вроде `.pdd` Medtronic) — коннектор их **не касается**. Для них в Claude Desktop есть встроенный Filesystem-коннектор. Claude видит оба источника одновременно — это и есть гибридная приватность.

## Что я смогу делать

```
«какие у меня тренды веса за последний месяц?»
«что я ел вчера, посчитай средний КБЖУ за неделю»
«запиши что я съел омлет с тостом на 450 ккал»
«посмотри давление за 2 недели — есть паттерн?»
«прочитай мой КПТ-дневник (локальная папка) и сравни с днями, когда я ел больше нормы»
```

Последний пример — **главная фишка**: Botkin MCP видит «калории 2800 ккал 18 мая», а Filesystem-коннектор видит «КПТ-запись 18.05: тяжёлый разговор, тревожность 8/10» — и Claude связывает, не отправляя приватный дневник на сервер.

## Как подключить

### Шаг 1 — получи токен в боте

В чате с [@Botkin_md_bot](https://t.me/Botkin_md_bot) напиши:

```
/connect_mcp
```

Бот спросит какой тип доступа нужен:

- **Полный (rw)** — чтение и запись. Для своего личного Claude.
- **Только чтение (ro)** — можно отдать врачу или другому специалисту. Он увидит данные, но ничего не изменит.

Выбери нужный → бот выдаст строку вида:
```
pat_895655_a1b2c3d4e5f6...
```

Это **твой личный токен**. Не публикуй его.

> **Управление токенами** — `/my_connections`: посмотреть все свои токены, отозвать ненужный.

### Шаг 2 — скачай коннектор

Скачай файл `botkin-connector.mcpb` — найдёшь его в [Releases репозитория](https://github.com/botkin-health/Botkin/releases) или попроси Александра.

### Шаг 3 — установи в Claude Desktop

1. Открой **Claude Desktop** → Settings → Extensions
2. Нажми «Install extension» и выбери скачанный файл `botkin-connector.mcpb`
3. В поле **«Токен Botkin»** вставь токен из шага 1
4. Поле **«Сервер API»** оставь как есть (`https://health.orangegate.cc`) — менять не нужно

### Шаг 4 — проверь

В новом чате напиши:

```
покажи мой профиль из Botkin
```

Claude ответит с твоими данными. Если не работает — см. «Если что-то не работает» ниже.

## Технический путь (для тех, кто хочет руками)

Если `.mcpb` недоступен, можно добавить коннектор вручную через `claude_desktop_config.json`:

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "botkin": {
      "command": "python",
      "args": ["/path/to/scripts/mcp/botkin_pat_mcp.py"],
      "env": {
        "BOTKIN_PAT": "pat_895655_a1b2c3d4...",
        "BOTKIN_API_BASE": "https://health.orangegate.cc"
      }
    }
  }
}
```

Зависимости: `pip install mcp requests` (Python 3.11+).

После сохранения — **Cmd+Q** и снова открыть Claude Desktop.

## Доступные инструменты

После подключения Claude получает ~10 инструментов и сам решает когда их вызвать.

### Чтение

| Инструмент | Что возвращает |
|---|---|
| `get_user_profile` | Имя, возраст, онбординг-данные |
| `get_day_summary` | Сводка дня: питание, активность, давление |
| `get_recent_meals` | Приёмы пищи с КБЖУ (до 7 дней) |
| `get_weight_history` | История веса и состава тела |
| `get_recent_bp` | История давления |
| `get_recent_workouts` | Последние тренировки |
| `get_recent_biomarkers` | Биомаркеры из анализов |
| `botkin_api` | Escape-hatch: произвольный GET к любому `/api/agent/*` endpoint |

### Запись (только rw-токен)

| Инструмент | Что делает |
|---|---|
| `log_meal_text` | Записать приём пищи текстом («гречка с курицей 300г») |
| `log_bp` | Записать измерение давления |

Запись с ro-токеном → Claude получит ошибку 403 и скажет тебе об этом понятно.

## Если что-то не работает

| Проблема | Решение |
|---|---|
| Claude говорит «не вижу Botkin tools» | Не перезапустил после установки. Полный **Cmd+Q** + открыть снова. |
| Ошибка «PAT недействителен» | Токен неверный или отозван. Выпусти новый: `/connect_mcp` → бот создаст свежий. |
| Ошибка «токен только для чтения» | Ты просишь записать данные, но выдал ro-токен. Выпусти rw-токен. |
| Хочу видеть локальные файлы тоже | Добавь Filesystem-коннектор. В Claude Desktop: Settings → Extensions → встроенный «Filesystem». |
| Хочу поделиться данными с врачом | `/connect_mcp` → выбери «только чтение (ro)» → отдай строку токена врачу. |
| Боюсь что токен утечёт | `/my_connections` → отозвать. Старый перестанет работать мгновенно. |

## Безопасность

- Каждый токен видит данные **только одного пользователя** — RLS на уровне БД, чужие данные недоступны физически
- rw-токен — только для себя; ro-токен — для врача или специалиста, он не может ничего изменить
- При потере токена: `/my_connections` → отозвать → выпустить новый
- JWT внутри коннектора живёт 5 минут, PAT хранится только у тебя и в keychain системы
