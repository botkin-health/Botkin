# 🔌 Claude Desktop MCP — личный доступ к своим данным

> Для **коллабораторов** проекта (Олег, Ника, Андрей и далее): подключи Botkin MCP к своему Claude Desktop Code, чтобы Claude видел твои данные и мог обсуждать твоё здоровье в любой сессии.

## Что такое MCP

**MCP** (Model Context Protocol) — стандартный способ дать AI-агенту (как Claude Desktop) доступ к внешним данным. Это как «адаптер» между Claude и серверами/файлами/API.

**Botkin MCP** — наш сервер, который даёт твоему Claude Desktop:
- 📊 Чтение твоих данных (питание за неделю, добавки, вес, давление, анализы)
- ✍️ Запись (можешь логировать еду прямо из чата с Claude)
- 📂 Доступ к твоей shared Google Drive папке (если есть)
- 📚 Общий longevity-KB (биомаркеры, протоколы Attia/Sinclair и т.д.)

Параллельно у тебя могут быть **другие MCP** в Claude Desktop:
- `filesystem` — даёт Claude доступ к локальным папкам твоего мака (там может лежать твой **приватный** КПТ-дневник или intimate notes)
- Любые другие что у тебя есть

Claude видит **и сервер, и локалку одновременно** — и может отвечать с учётом обоих источников. **Сервер не видит локальные приватные файлы.**

## Что я смогу делать

Примеры реальных запросов в чате с Claude:

- «какие у меня тренды веса за последний месяц»
- «что я ел вчера, посчитай средний КБЖУ за неделю»
- «у меня уровень ферритина 112 — это норма для моего возраста?»
- «дай советы по моим последним анализам, что подсветить врачу»
- «запиши что я съел омлет с тостом на 450 ккал»
- «прочитай мой КПТ-дневник за неделю (локальный) и сравни с моими данными о сне/настроении — есть корреляция?»

Последний пример самый интересный — это **главная фишка**. Сервер видит «sleep 6.5h, stress high», твой Claude видит «в дневнике 18.05: тяжёлый разговор с мамой, тревожность 7/10» — и связывает.

## Как подключить (5 минут)

### Шаг 1 — получи MCP-токен в боте

В чате с [@Botkin_md_bot](https://t.me/Botkin_md_bot) напиши:

```
/mcp_token
```

Бот пришлёт строку вида:
```
hvmcp_895655_x1y2z3a4b5c6...
```

Это **твой персональный ключ**. Не делись им.

### Шаг 2 — открой Claude Desktop config

Зависит от системы:

**macOS:**
```
~/Library/Application Support/Claude/claude_desktop_config.json
```

**Windows:**
```
%APPDATA%\Claude\claude_desktop_config.json
```

Открой в любом текстовом редакторе (VS Code, TextEdit, Notepad).

### Шаг 3 — добавь Botkin MCP

В файле найди (или создай) объект `mcpServers` и добавь туда `healthvault`:

```json
{
  "mcpServers": {
    "healthvault": {
      "type": "http",
      "url": "https://mcp.health.orangegate.cc",
      "headers": {
        "Authorization": "Bearer hvmcp_895655_x1y2z3a4b5c6..."
      }
    }
  }
}
```

Замени токен на свой из шага 1.

**Если у тебя уже есть другие MCP** (например, filesystem) — просто добавь `healthvault` в тот же объект `mcpServers`:

```json
{
  "mcpServers": {
    "filesystem": { "...твоя конфигурация..." },
    "healthvault": {
      "type": "http",
      "url": "https://mcp.health.orangegate.cc",
      "headers": {
        "Authorization": "Bearer hvmcp_895655_x1y2z3a4b5c6..."
      }
    }
  }
}
```

### Шаг 4 — перезапусти Claude Desktop

После сохранения файла **полностью закрой** Claude Desktop (Cmd+Q на маке) и открой снова. Только перезапуск перечитывает конфиг.

### Шаг 5 — проверь

В новом чате напиши:
```
покажи мой профиль из Botkin
```

Claude должен ответить с твоими данными: имя, возраст, текущие лекарства, основные показатели.

Если не работает — см. раздел «Если что-то не работает» ниже.

## Какие tools доступны

После подключения у Claude появляется ~12 инструментов. Он сам решает когда какой вызвать. Список:

### Чтение

| Tool | Что возвращает |
|---|---|
| `get_my_profile` | Имя, возраст, диагнозы, лекарства, цели |
| `get_my_meals(days=7)` | Приёмы пищи с КБЖУ |
| `get_my_supplements(days=7)` | Лог добавок и лекарств |
| `get_my_weights(days=90)` | История веса/жира/мышц |
| `get_my_bp(days=30)` | История давления |
| `get_my_activity(days=30)` | Шаги, сон, HRV, активные ккал |
| `get_my_dashboard_summary` | Текстовая сводка с дашборда |
| `read_my_kb` | Содержимое твоего knowledge_base.json |
| `query_my_kb(query)` | Поиск в твоём KB |
| `get_longevity_reference(topic)` | Содержимое общего longevity KB (apo-b, vit-d, sleep, etc) |

### Запись

| Tool | Что делает |
|---|---|
| `log_meal` | Записать приём пищи |
| `log_supplement` | Записать приём добавки/лекарства |
| `log_bp` | Записать измерение давления |
| `log_weight` | Записать вес |
| `update_my_profile` | Обновить smoking_status, chronic_conditions и т.д. |

## Если что-то не работает

| Проблема | Решение |
|---|---|
| Claude говорит «не вижу Botkin tools» | Не перезапустил после правки конфига. Полный Cmd+Q + открыть снова. |
| Claude вызывает tool, получает 401 | Токен неверный или истёк. Сделай `/mcp_token rotate` в боте, обнови в config |
| `claude_desktop_config.json` не существует | Создай файл руками, добавь блок выше. Путь — выше в Шаге 2 |
| Я хочу одновременно видеть свои файлы на маке | Добавь второй MCP — `filesystem`, его конфиг описан в [официальной документации](https://modelcontextprotocol.io/docs/quickstart/user) |
| Боюсь что токен утечёт через git | **Не коммить `claude_desktop_config.json` в публичные репо.** Если коммитнул случайно — `/mcp_token rotate` |

## Безопасность

- Токен видит **только твои** данные. Чужие не покажет — на сервере фильтр по `user_id`
- Если потерял или хочешь поменять — `/mcp_token rotate` в боте, старый перестанет работать
- Если кажется что кто-то украл доступ — `/mcp_token revoke`, тогда токен отзывается насовсем (новый выдашь когда захочешь)

Подробнее — в [Безопасность](./security.md).
