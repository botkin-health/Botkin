### 1. Дашборд (Mission Control) — теперь живой

Самое главное из нового — я собрал полноценный персональный дашборд. Это HTML-страница, которую можно открыть в любом браузере и которая показывает все мои данные здоровья за последние 90 дней в одном месте.

**Посмотри прямо сейчас:** https://health.orangegate.cc/mc/21e58962-0368-457f-8a5d-93c66762e156

Что там есть:
- **Тело** — вес 76.75 кг (−6.25 кг с января), % жира по биоимпедансу, динамика по неделям
- **Питание** — среднесуточные ккал/БЖУ, дефицит/профицит, топ продуктов, алкоголь
- **Сон** — длительность, фазы (REM/deep/light), тренды
- **Сердце** — ЧСС покоя, HRV, Body Battery, стресс-индекс
- **Анализы крови** — ключевые биомаркеры (LDL, витамин D, HbA1c, тестостерон и т.д.) с цветовой индикацией норм
- **Давление** — динамика систолы/диастолы
- **Воздух дома** — CO₂, температура, влажность спальни (Netatmo)
- **Спорт** — новый большой блок, подробнее ниже

Технически это `telegram-bot/dashboard_generator.py` + `mc_template.html`. Данные берутся из PostgreSQL и JSON-файлов, HTML генерируется один раз при запросе, кэшируется. Никакого React/Vue — чистый JS в одном файле.

### 2. Блок спортивной аналитики — по канонам Seiler и Attia

Это самая интересная новинка технически. Я добавил большой блок с анализом тренировок, основанный на трёх научных подходах:

**Поляризованная модель (Seiler, 80/20):** профессиональные выносливостные атлеты тратят ~80% времени в лёгких зонах (Z1+Z2), ~5% в переходной (Z3) и ~15-20% в высокоинтенсивных (Z4+Z5). Модель подтверждена от элитных марафонцев до любителей. Я смотрю на свои данные через эту призму.

**Цели Attia (Peter Attia, "Outlive"):**
- Зона 2 (аэробная база): 150+ минут в неделю — фундамент митохондриального здоровья
- Z4+Z5 (острые интервалы): 16+ минут в неделю — VO2max

**Острая:Хроническая нагрузка (WHOOP/ATL:CTL):** 7-дневная нагрузка ÷ 90-дневная средняя. Сладкое пятно 0.8–1.3. Выше 1.5 — риск перегрузки и травмы.

Что показывает блок:
- 4 KPI карточки: тренировок/нед, минуты Z2/нед, минуты ВИИТ/нед, Острая/Хроническая нагрузка
- Постоянная легенда HR-зон (всегда видна, не нужно помнить)
- Пирамида: каждая из 12-13 недель в окне 90 дней — как распределялась нагрузка по зонам
- Список последних 14 дней тренировок с HR-зонами в виде цветовой полосы
- Вердикт: поляризованный / пирамидальный / ловушка Z3 / всё лёгкое / перегруз
- Конкретные рекомендации с расчётом дефицита

### 3. Phantom workouts — детективная работа с ЧСС

Интересный технический кейс. Октябрь-декабрь 2025 я тренировался, но не логировал в Garmin часть занятий — просто не выбирал тип активности на часах. Потерял ~15-20 тренировок из статистики.

Garmin хранит минутный ЧСС за ~6 последних месяцев. Написал скрипт `detect_phantom_workouts.py`, который:
1. Находит дни без залогированной тренировки
2. Скачивает минутный ЧСС за этот день
3. Применяет скользящее окно 65 минут — ищет блок с ≥35% точек с ЧСС ≥ 90 bpm (это CrossFit/силовой паттерн — не непрерывный подъём, а чередование подъёмов и пауз между сетами)
4. Вычисляет HR-зоны, оценивает нагрузку
5. Сохраняет как `is_synthesized=True` — в аналитику входят, в Garmin не загружаются

Результат: 11 восстановленных тренировок за декабрь 2025. 2 из них с confidence=high (maxHR 132 и 144 bpm — очевидный факт), остальные medium. В дашборде отображаются с ψ-маркером.

Ноябрь и октябрь восстановить не удалось — Garmin уже удалил минутные данные (>6 месяцев).

## TL;DR ⭐

**Проблема.** В 48 лет хочется жить хорошо и долго. Есть 18+ источников данных о здоровье: Garmin, iPhone, Apple Health, умные весы, домашняя метеостанция, квартальные анализы крови, МРТ/УЗИ архив за 15 лет. Все они размазаны по разным приложениям, экспортам и PDF-кам. Ни одно коммерческое решение не сводит это вместе.

**Решение — два связанных проекта:**
- **HealthVault** — методология: как собрать в одно место, как структурировать «медицинскую карту» в JSON, какие корреляции искать, как вести диалог с AI про своё здоровье. Это skills для Claude Code + схема `knowledge_base.json` + промпты. Код минимальный — это **фреймворк для того, кто готов вести дисциплинированную работу со своим здоровьем**.
- **NutriLogBot** — практический Telegram-бот: фотографируешь еду / диктуешь голосом → LLM распознаёт КБЖУ → пишется в Postgres → видишь в мини-приложении внутри Telegram + в дашборде.

**Что работает с января 2026:** бот в проде, 18 потоков данных синкаются регулярно, дашборд живой и публичный (см. ссылку выше), жена пользуется с марта. Сбросил 6.25 кг за 3.5 месяца.

## Claude skills — как это устроено

В `~/.claude/skills/` лежат папки со Skill.md — это инструкции для Claude Code. Каждый skill — slash-команда:

| Команда | Что делает |
|---|---|
| `/sync` | Тянет все 18 источников данных, показывает таблицу актуальности |
| `/dashboard` | Сводка: какие потоки актуальны, где провалы |
| `/chart` | Генерирует PNG с динамикой веса/калорий/тренировок |
| `/cleanup` | __pycache__, git commit/push, бэкап БД, ротация бэкапов |

Добавить skill — создать папку в `~/.claude/skills/имя/SKILL.md` и написать туда markdown-инструкцию. Claude Code подхватывает автоматически. Это очень мощный паттерн — можно сделать `/morning-brief`, `/weekly-report`, `/compare-labs` под свои нужды.

## Семейная база знаний

Каждый человек — отдельная папка в Google Drive с `knowledge_base.json` и `PROFILE.md`:

```json
{
  "blood_tests": [{
    "date": "2026-03-12",
    "lab": "Инвитро",
    "markers": {
      "ldl": 3.1,
      "hdl": 1.4,
      "vitamin_d": 35.6,
      "hba1c": 5.2
    },
    "summary": "LDL исторический минимум. Витамин D наконец в оптимуме."
  }]
}
```

PDF-анализы — архив. JSON — источник истины, по которому Claude отвечает на вопросы. Когда я спрашиваю «как менялся мой LDL за 5 лет» — он читает JSON и строит ответ со ссылками на конкретные PDF.

## Корреляции, которые нашла система

Это то, что не находится без сведения всех данных в одно место:

- **Сон и HRV:** стабильная связь «поздний отход ко сну → HRV ниже на следующий день на 8–12 мс». Теперь стараюсь ложиться до 23:30.
- **Вес и питание:** точная связь «недельный дефицит калорий → −0.4 кг за неделю» подтверждена на 3 месяцах. Используется для управления темпом похудения.
- **CO₂ и АД:** при CO₂ в спальне >1200 ppm (Netatmo) утреннее давление выше на ~5 мм рт. ст. Стали проветривать спальню перед сном.
- **Тренировки:** Обнаружил, что называл CrossFit «HIIT» в Garmin — но реальные пиковые зоны (Z4+Z5) составляли <10% тренировки. Переименовал 25+ исторических активностей через API.

## Два пути для тебя ⭐

### Путь 1. Пользователь на моём сервере

Бот `@HealthVault_bot` теперь с открытой регистрацией — пиши `/start` прямо сейчас. Через онбординг введёшь имя, рост, вес, цель. Твои данные изолированы в Postgres по user_id — я вижу только свои.

Что получишь сразу: пищевой дневник (фото / голос / текст), добавки, Telegram мини-аппа с графиками.
Что не получишь сразу: свой дашборд (пока генерируется только мой), Garmin-интеграция (нужны твои логин/пароль), Netatmo (нужна своя метеостанция).

### Путь 2. Своя копия (рекомендую если интересно технически)

Клонируешь репо → настраиваешь `.env` → `docker-compose up`. Нужно:
- VPS (Hetzner/DigitalOcean/любой, €5–10/мес)
- Свой Telegram-бот через @BotFather
- OpenAI API ключ (основа распознавания еды, ~$5-10/мес при регулярном использовании)
- Опционально: Garmin (логин/пароль достаточно), Zepp/Xiaomi весы, Netatmo

Главный плюс Пути 2: делаешь что хочешь, присылаешь PR — вместе развиваем одну кодовую базу.

### Путь 3 (мой любимый) — гибридный

Начинаешь с Пути 1 (пишешь еду в мой бот, привыкаешь к паттерну). Параллельно ставишь себе Claude Code и начинаешь строить свою `knowledge_base.json` — просто структурированный JSON с твоими анализами и устройствами. Через месяц-два когда репо будет чище — форкаешь и поднимаешь своё.
