# Research: ChatGPT Health (OpenAI) vs Botkin — фичевое сравнение

Дата: 2026-05-30 · По запросу: «изучи всё что у них есть, сравни по фичам — что есть и схоже, что анонсируют»

## TL;DR

- **ChatGPT Health запущен 7 янв 2026** — отдельное «здоровье»-пространство в ChatGPT: подключение медкарт (EHR/патиент-порталы) + wellness-приложений (Apple Health, MyFitnessPal, Function, Peloton, Weight Watchers, AllTrails, Instacart), интерпретация анализов, подготовка к визиту врача, ответы с цитатами из peer-reviewed источников.
- **Гео — главный разрыв в нашу пользу:** killer-фича (медкарты/EHR) и часть приложений — **только США**. Запуск шёл **в обход EEA, Швейцарии и UK** (они исключены). Россия/СНГ — не в планах вообще. Планы расширения: Индия, Бразилия, Мексика, Филиппины.
- **Что у них сильнее Botkin:** интеграция с реальными медкартами/EHR клиник (US), партнёрство с Apple Health «из коробки», датасеты с врачебными бенчмарками, цитаты из миллионов исследований, бренд + 230M запросов о здоровье/нед.
- **Где Botkin структурно защитим:** приватность/own-your-data, open-source/self-host, **не-US рынок (вкл. RU/EU)**, семейный/caregiver-сценарий (вести родителей), истинная мультиисточниковая лонгитюдная аналитика (Garmin+Zepp+Netatmo+лабы+генетика+скринтайм), биовозраст/longevity-модели (PhenoAge/SCORE2/ASCVD).
- **Их роадмап:** глубже EHR, мультимодальный анализ снимков (рентген/МРТ), предиктивная аналитика. Это догонит часть наших возможностей в US, но не закрывает приватность/гео/open-source.

## Контекст

Боткин — приватная семейная open-source платформа трекинга здоровья (Telegram + сервер + BotkinClaw на Opus 4.8). Вопрос возник после разговора с Женей Смородниковой (Welltory): «ни один инвестор не вложит — не объяснишь чем ты лучше OpenAI Health». Цель отчёта — на фактах понять, где конкуренция реальна, а где у Боткина защитимый пятачок.

## Что такое ChatGPT Health (факты на янв 2026)

Два разных продукта запущены одновременно:

1. **ChatGPT Health (consumer)** — 7 янв 2026. Отдельное sandbox-пространство внутри ChatGPT.
2. **ChatGPT for Healthcare / OpenAI for Healthcare (enterprise)** — 8 янв 2026, для клиник (AdventHealth, Cedars-Sinai, Boston Children's, MSK, Stanford, UCSF). Clinical evidence search, care pathways, referral letters, HIPAA. **Это B2B, не наш сегмент** — но показывает, куда идёт компания.

### Фичи consumer-версии (что реально есть)

| Фича | Детали |
|---|---|
| Подключение медкарт | EHR / патиент-порталы клиник напрямую — **US only** |
| Wellness-приложения | Apple Health, Function, MyFitnessPal, Weight Watchers, AllTrails, Instacart, Peloton (часть — US only; Apple Health требует iOS) |
| Интерпретация анализов | Разбор lab results, истории болезни |
| Подготовка к визиту | Сформулировать вопросы врачу, разобрать визит-саммари |
| Диета/тренировки | Советы по питанию и нагрузке |
| Страховка | Сравнить опции страховки по паттернам обращений (очень US-специфично) |
| Цитаты-источники | Ответы со ссылками на peer-reviewed исследования, клин. гайдлайны (заголовок/журнал/дата) |
| Приватность | Спец-шифрование + изоляция, отдельный memory-store, **не используется для обучения foundation-моделей** |
| Дисклеймер | НЕ для диагностики/лечения |

### Гео и доступ (критично!)

- **Медкарты/EHR + часть приложений — только США.**
- Запуск для Free/Go/Plus/Pro **за пределами EEA, Швейцарии и UK** — т.е. эти регионы **исключены** (привет, регуляторика GDPR/EU AI Act).
- Планы расширения: **Индия, Бразилия, Мексика, Филиппины** (где плохой доступ к врачам).
- **Россия/СНГ — не упоминаются нигде.** Для Боткина это значит: на его текущем рынке (RU + русскоязычные + EU-приватники) прямого конкурента-гиганта **нет**.

### Роадмап (анонсировано «скоро»)

- Глубже **EHR-интеграции**
- **Мультимодальный анализ изображений** (медицинские снимки — рентген, МРТ, дерматология)
- **Предиктивная аналитика**
- Раскатка на всех web + iOS «в ближайшие недели»

### Риски/критика (из прессы)

- Галлюцинации и biases в мед-ответах; иск о wrongful death к OpenAI; тревога регуляторов по mental health.
- Приватность: данные всё равно на серверах OpenAI; эксперты отмечают «люди не понимают как их данные используются».

## Фичевое сравнение: ChatGPT Health vs Botkin

### ✅ Что есть у обоих (схоже)

| Фича | ChatGPT Health | Botkin |
|---|---|---|
| Conversational AI-агент о здоровье | ✅ (GPT) | ✅ BotkinClaw (Opus 4.8) |
| Интерпретация анализов крови | ✅ | ✅ KB + агент |
| Apple Health интеграция | ✅ (нативно, iOS) | ✅ через Health Auto Export |
| Питание/добавки трекинг | ✅ (MyFitnessPal и т.п.) | ✅ нативно (фото/текст еды) |
| Память диалога о здоровье | ✅ изолированная | ✅ agent_conversations |
| Ответы с опорой на данные юзера | ✅ | ✅ tools (JWT+RLS) |

### 🔴 Что есть у ChatGPT Health, чего НЕТ у Botkin

| Фича | Почему важно | Реалистично ли догнать |
|---|---|---|
| **Прямая интеграция с EHR/медкартами клиник** | Автоподтяжка истории болезни без ручного ввода | ❌ Только US, нужны контракты с провайдерами. Для RU — неактуально |
| **Цитаты из миллионов исследований** | Доверие, доказательность | 🟡 Частично — можно прикрутить PubMed/RAG, но не их масштаб |
| **Датасеты с врачебными бенчмарками** | Качество мед-ответов | ❌ Не воспроизвести в одиночку |
| **Бренд + дистрибуция** | 230M запросов/нед | ❌ Несопоставимо |
| **Мультимодальный анализ снимков** (роадмап) | Рентген/МРТ/кожа | 🟡 Через API моделей — но не приоритет |

### 🟢 Что есть у Botkin, чего НЕТ (или структурно не будет) у ChatGPT Health

| Фича | Botkin | ChatGPT Health |
|---|---|---|
| **Own-your-data / self-host** | ✅ свой сервер + open-source | ❌ данные на OpenAI |
| **Гибридная приватность** (локальный приватный слой через MCP) | ✅ | ❌ |
| **Open-source** | ✅ | ❌ структурно никогда |
| **Не-US рынок (RU/СНГ/EU)** | ✅ работает везде | ❌ US-first, EEA/UK/CH исключены, RU не в планах |
| **Семейный/caregiver-режим** (вести родителей, мульти-профиль с RLS) | ✅ cohort-роли | ❌ персональный, про «себя» |
| **Истинная мультиисточниковость + лонгитюд** | ✅ Garmin+Zepp+Netatmo+Apple+лабы+генетика+скринтайм+погода+среда | 🟡 в основном снимок из Apple Health/приложений, не глубокий лонгитюд по многим девайсам |
| **Биовозраст / longevity-модели** | ✅ PhenoAge, LE8, SCORE2, ASCVD, Attia | ❌ нет специализированных |
| **Континуальная среда** (климат спальни, погода ↔ сон/HRV) | ✅ Netatmo | ❌ |
| **Telegram-нативность** (там где живёт RU/CIS-аудитория) | ✅ | ❌ только ChatGPT app/web |

## Рекомендация

**Конкурировать в лоб — нет. Занять защитимый пятачок — да, и пятачок реально хороший.**

Три факта меняют картину против Жениного «не объяснишь чем ты лучше»:

1. **Гео.** OpenAI Health killer-фичей (медкарты) работает только в США, и сознательно **исключил EEA/UK/CH**, а RU/СНГ не в планах. На рынке Боткина (русскоязычные + EU-приватники + caregiver-сценарии) **прямого конкурента-гиганта физически нет**.
2. **Приватность как продукт, а не оговорка.** «Твои интимные данные о здоровье НЕ уезжают в OpenAI, код открыт, можешь хостить сам» — это не маркетинг, а структурное отличие, которое OpenAI не может скопировать без слома своей модели.
3. **Семья/родители.** ChatGPT Health — про «себя». Боткин уже ведёт маму и папу. «Sandwich generation» (40-55 лет, ведут стареющих родителей) — огромная недообслуженная ниша, и это твой личный реальный use-case.

**Позиционирование, которое работает:** *«Приватный open-source AI-аналитик здоровья для семьи — данные остаются у тебя, работает там, где ChatGPT Health не работает»*. Не «убийца ChatGPT», а «то, чем ChatGPT Health не станет».

**Чего НЕ делать:** не пытаться повторить EHR-интеграции/врачебные датасеты/цитатный масштаб — там не выиграть, и на RU-рынке это и не нужно.

## Следующие шаги для Botkin

- [ ] Зафиксировать позиционирование «privacy-first + family + non-US» в лендинге botkin.health (сейчас там общий месседж)
- [ ] Усилить caregiver-сценарий как явную фичу: мульти-профиль, «дашборд родителя для ребёнка» (частично есть через cohort — оформить как продукт)
- [ ] Контент-аргумент «почему не отдавать здоровье в OpenAI» (приватность as feature) — для будущего онбординга/лендинга
- [ ] Опционально: лёгкий PubMed/RAG-слой для цитат в ответах BotkinClaw (закрыть разрыв по доказательности дёшево)
- [ ] НЕ инвестировать в: EHR-интеграции, мультимодальный анализ снимков (низкий приоритет на RU-рынке)
