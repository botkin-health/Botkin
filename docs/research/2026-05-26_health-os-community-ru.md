## 2. Founder OS #23 и #24 — реальные артефакты

**Что это на самом деле:** еженедельные открытые сессии AI Mindset {space} (FOS = Forum Open Space). Страница спринта: [space-health-sprint.netlify.app](https://space-health-sprint.netlify.app/)

### FOS #23 (5 марта 2026) — personal health stack
**Дмитрий Твердохлебов** (ex-Яндекс Senior PM, ментор Сколково MBA):
- Построил медицинский дашборд из ~4GB данных (6.4M записей) Apple Health
- Без программирования, через Claude Agent
- Публичных артефактов (GitHub, блог) не найдено

**Константин Попов:**
- 12 лет дневниковых записей + биометрия + AI-корреляции
- Пример: связь паттернов питания и симптомов ГЭРБ

### FOS #24 (12 марта 2026) — геном и Health Portal
**Григорий Земсков:**
- 8-шаговый пайплайн анализа полного генома через Claude Code
- Импутация → аннотация → фармакогенетика → AI-скоры
- 1800+ находок в персональном отчёте

**Тоня Жукова:**
- «Health Portal» с 15 AI-агентами для мониторинга липидного профиля и кардиоваскулярных рисков
- Интеграция: геном + лабораторные анализы
- Публичных репо не найдено

**Важно:** все это личные пет-проекты, показанные в рамках сессии. Ни у кого нет открытого репо — люди строили для себя. Это подтверждает потенциал Botkin как готовой open-source базы для таких же энтузиастов.

## 3. Илья Гиндин / «OpenHealth-framework»

**Найдено:** только косвенно. Наиболее близкий публичный проект с похожим названием:

### [OpenHealthForAll/open-health](https://github.com/OpenHealthForAll/open-health) — ⭐3.9k
**Язык:** TypeScript (99%), Next.js
**Активность:** последний релиз v0.2.0 от 21 февраля 2025, 275 коммитов, 417 форков
**Что делает:** AI Health Assistant — приватный локальный ассистент на основе LLM (LLaMA/GPT/Claude/Gemini), импортирует медданные из разных источников, развёртывается через Docker
**Связь с Гиндиным:** не установлена. Авторы проекта — org OpenHealthForAll, конкретные имена не раскрыты

**Итог:** Илья Гиндин с «OpenHealth-framework» публично не идентифицирован. Либо это упоминание из закрытого Telegram-чата, либо ранняя стадия без публичного репо.

**Применимость open-health к Botkin:**
- Архитектурно похоже: LLM + личные данные + приватность
- Отличие Botkin: Telegram-интерфейс, PostgreSQL бэкенд, семейный multi-user, Garmin sync — то, чего у open-health нет
