# 0008. Отчёт для врача: содержание по IPS, артефакт — человекочитаемый PDF

**Status:** Accepted
**Date:** 2026-07-08
**Deciders:** Игорь Лысковский (реализация), Александр Лысковский (постановка — п.6 роадмапа)
**Context:** Пользователю нужно передать свои данные из Botkin врачу на приёме. До этого
был только HTML-дашборд/отчёт по ссылке (`/report` → `/r/{token}`), заточенный под
самомониторинг: переслать врачу нечего, порядок блоков не клинический. Нужен формат,
понятный врачу, и надёжный способ его отдать.

## Решение

**Модель содержания** отчёта = секции **International Patient Summary (ISO 27269)** в
клиническом порядке (Проблемы → Аллергии → Лекарства → Результаты → Витальные → Образ жизни).
**Артефакт** = человекочитаемый **PDF**, который бот генерит и отправляет пользователю
Telegram-документом (пользователь пересылает врачу). PDF-рендер — **weasyprint** (HTML→PDF,
server-side). Машинный FHIR/CDA/СЭМД-экспорт — **вне scope**.

## Альтернативы

- **Сертифицированный СЭМД через ЕГИСЗ (РФ) или полный FHIR/CDA IPS** — отвергнуто: требует
  аккредитации МИС, ЭЦП врача/МО, гос-интеграции. Это инфраструктура мед-организаций, а не
  wellness-приложения. Берём только *таксономию секций* IPS, не сам обмен.
- **Отдавать существующий дашборд `/mc/` / отчёт `/r/`** — отвергнуто как основной путь: он под
  самомониторинг (не клинический порядок), и он же расшарен врачу как лайв-страница. Врачу нужен
  снимок-документ, а не интерактивный дашборд.
- **PDF через headless-браузер (playwright/chromium)** — отвергнуто: тяжёлый браузер в контейнере
  (размер образа + ресурсы) ради статичного документа. weasyprint достаточно.
- **Программный PDF (reportlab/fpdf)** — отвергнуто: выбрасывает готовый HTML-пайплайн (дублирование).
- **Веб-ссылка вместо файла** — отвергнуто по UX: пользователю удобнее переслать готовый PDF-файл
  в чате, чем ссылку (решение Игоря, 08.07).

## Последствия

- **Позитивные:** врач получает знакомый по структуре снимок (IPS — международный стандарт состава);
  переиспользуем данные существующего пайплайна (`dashboard_generator`, `health_reports`,
  `aggregate_biomarkers` с канонизацией kb_schema); единый helper доставки `send_doctor_report_to_chat`
  для кнопки мини-аппа и (в follow-up) агент-тула и команды.
- **Негативные / trade-offs:** weasyprint тянет системные libs (pango/cairo/gdk-pixbuf) в Docker-образ;
  PDF-рендер не прогоняется в CI/локали без GTK-libs → верификация PDF на on-device E2E. Три
  IPS-обязательные секции (проблемы/аллергии/лекарства) у Botkin слабые (только онбординг) — показываем
  их честно с пометкой «со слов пользователя».
- **Что НЕ делать (anti-pattern):** не пытаться эмитить сертифицированный СЭМД/CDA из wellness-бота;
  не класть кнопку «Экспорт» внутрь iframe `/mc/` (он расшарен врачу) — она живёт в chrome мини-аппа;
  не выдавать отсутствие данных за норму.
