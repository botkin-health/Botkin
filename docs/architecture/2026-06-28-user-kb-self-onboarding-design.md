# Дизайн: Самостоятельное подключение KB пользователем

**Дата:** 2026-06-28  
**Статус:** approved  
**Контекст:** сейчас KB подключается вручную через `config/users_private.py` + SSH-синк администратором. Цель — дать каждому новому пользователю самостоятельный путь.

---

## Выбранный подход

**Команда `/doc` + файловый архив на сервере.**

Пользователь присылает медицинский документ боту в любой момент, Claude извлекает данные, пользователь подтверждает, файл и данные сохраняются. Никакого участия администратора.

---

## Пользовательский флоу

**Точка входа:** команда `/doc` в Telegram.

```
1. /doc
   → «Пришли PDF, фото или скан анализа/заключения.»

2. Пользователь присылает файл
   → Бот: «⏳ Читаю...»
   → Claude читает через vision (фото/скан) или PDF-extraction

3. Бот показывает превью:
   «Нашёл в документе:
    • Дата: 13.04.2026
    • Лаборатория: KDL
    • Гемоглобин: 165 г/л
    • Ферритин: 112 нг/мл
    • АЛТ: 24.3 Ед/л
    [Сохранить ✅] [Отмена ❌]»

4а. «Сохранить»
    → файл → /data/uploads/<user_id>/
    → данные → kb_<user_id>.json documents[]
    → «✅ Сохранено. Можешь прислать ещё или /doc в любой момент.»

4б. «Отмена»
    → ничего не сохраняется, файл удаляется
```

**Упоминание в онбординге:** в финальном сообщении (после шага wearables) добавляется строка:
> «📂 Есть анализы или заключения врачей? Пришли их мне командой /doc — буду помнить твою историю здоровья.»

После «✅ Сохранено» бот предлагает прислать ещё — пользователь может загрузить батч из нескольких документов подряд без повторного `/doc`.

## Хранение

**Файлы на сервере:**

```
/opt/botkin/data/uploads/
  └── <telegram_id>/
        ├── 2026-06-28_a1b2c3d4.pdf
        └── 2026-06-28_e5f6g7h8.jpg
```

Имя файла: `<дата>_<8 символов хеша>.<ext>`. Без PII в имени файла.

**Запись в `kb_<id>.json`:**

```json
{
  "documents": [
    {
      "added_at": "2026-06-28",
      "file": "2026-06-28_a1b2c3d4.pdf",
      "extracted": {
        "date": "2026-04-13",
        "laboratory": "KDL",
        "values": { "Hb": 165, "ferritin": 112, "ALT": 24.3 }
      },
      "user_confirmed": true
    }
  ]
}
```

Секция `documents[]` — отдельная от `blood_tests[]`. Агент читает её через существующий `/kb_value`. Postgres `blood_tests` не затрагивается на этом этапе — структурированный импорт остаётся отдельной операцией.

## Архитектура компонентов

| Компонент | Расположение | Что делает |
|-----------|-------------|------------|
| Handler `/doc` | `telegram-bot/handlers/doc_upload.py` | state machine: ждёт файл → вызывает экстрактор → показывает превью → обрабатывает кнопки |
| Claude-экстрактор | `core/health/doc_extractor.py` | принимает bytes + mime_type, возвращает `dict` с найденными значениями |
| Сохранение файла | `telegram-bot/webhook/agent_tools_api.py` | новый endpoint `POST /upload_doc` — принимает файл, пишет в `/data/uploads/<id>/` |
| Запись в KB | переиспользует `_resolve_user_kb_path()` | append в `documents[]`, атомарная замена файла через tmp |
| docker-compose | `docker-compose.prod.yml` | добавить bind-mount `./data/uploads:/app/data/uploads` |

Существующие компоненты (`_resolve_user_kb_path`, `data/kb/kb_<id>.json`) переиспользуются без изменений.

## Edge cases

**Claude не нашёл числовых значений:**
```
«Не нашёл числовых значений в документе.
Это всё равно можно сохранить как архив —
запомню что такой документ есть.
[Сохранить как архив 📁] [Отмена ❌]»
```
При «сохранить как архив» — `extracted: {}`, файл сохраняется. Агент может прочитать оригинал при разговоре.

**Ограничения на файлы:**
- Максимум 20 МБ (лимит Telegram Bot API)
- Форматы: PDF, JPG, PNG, HEIC (Telegram конвертирует HEIC → JPG автоматически)

**Файл вне контекста `/doc`:**
- Если пользователь шлёт PDF без команды `/doc` — бот не реагирует, обычный flow не прерывается (фото еды остаётся фото еды)

**Дублирование:**
- Claude в превью покажет те же значения что уже есть в KB — пользователь сам решает сохранять или нет

**Квота хранения:**
- На старте: без квоты
- Позже: лимит ~100 МБ на пользователя

## Что НЕ входит в scope

- Автоматический перенос данных из `documents[]` в `blood_tests` / Postgres — это отдельная задача
- Классификация типа документа (blood/UZI/MRI) — добавляется позже
- Приватный режим (файлы только на маке пользователя) — отдельный проект
- Квотирование и удаление файлов — отдельная задача
