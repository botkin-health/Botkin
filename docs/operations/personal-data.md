# Личные данные: что можно и нельзя класть в репозиторий

> Botkin — open-source проект (AGPL), но платформа работает с реальными медицинскими
> данными семьи. Этот документ — канонический чек-лист приватности. На него ссылаются
> `CLAUDE.md`, `docs/INDEX.md` и `scripts/mcp/botkin_mcp.py`.

---

## 🔒 Чего НЕ должно быть в публичном репо

- **Имена реальных пользователей.** Только cohort-термины: `owner` / `family` / `early_user` / `external`.
- **Диагнозы конкретных людей** — ни в коде, ни в доках, ни в комментариях, ни в примерах.
- **Идентификаторы:** `user_id`, `telegram_id`, email, телефоны, ИНН и т.п.
- **Полные пути к личной Google Drive папке** (структура `FamilyHealth/<Имя> — Здоровье/`).
- **Биомаркеры с цифрами конкретных людей** — даже «обезличенные» примеры с реальными значениями.
- **Личные цели здоровья:** анализы, био-возраст, целевой вес, планы тренировок.
- **Секреты:** токены, пароли, JWT-секреты, webhook-токены — только `.env` (в `.gitignore`).

## 📁 Где личные данные живут

| Тип данных | Место | В git? |
|---|---|---|
| Медицинские документы, KB, профили | `~/FamilyHealth/<user>/` (Google Drive) | ❌ никогда |
| Telegram ID, имена, диагнозы для AI-контекста | `~/.claude/CLAUDE.md` (приватный) | ❌ |
| Маппинг алиасов для MCP | `~/.botkin-personal.yaml` (см. ниже) | ❌ |
| Внутренние проектные доки с личным контекстом | `docs/projects/`, `docs/night-shift/`, `docs/superpowers/` | ❌ (в `.gitignore`) |
| Код, схемы, generic-документация | репозиторий | ✅ |

Перед коммитом нового файла в `docs/` — прогон по чек-листу выше. Если файл содержит
хоть один пункт — ему место в gitignored-папке или в `~/FamilyHealth/`.

## 🗺 Шаблон `~/.botkin-personal.yaml`

Приватный конфиг MCP-сервера (`scripts/mcp/botkin_mcp.py`): маппинг «алиас → папка
в `~/FamilyHealth/`». Лежит в домашней директории, НЕ в репозитории.

```yaml
# ~/.botkin-personal.yaml
person_map:
  self: "Имя Фамилия — Здоровье"      # обязательный алиас владельца
  wife: "Имя Фамилия — Здоровье"      # произвольные алиасы членов семьи
  son: "Имя Фамилия — Здоровье"
```

Если файла нет, MCP-сервер отвечает только на алиас `self`, беря папку из env
`BOTKIN_SELF_DIR`.

## 🧹 Если личные данные всё-таки попали в git

1. Удалить из рабочего дерева и закоммитить удаление.
2. Если это секрет (токен/пароль) — **ротировать немедленно**, история git публична.
3. Для имён/диагнозов в истории — переписать историю (`git filter-repo`) и force-push,
   либо признать утечку и зафиксировать в private-доке решение «оставляем».
